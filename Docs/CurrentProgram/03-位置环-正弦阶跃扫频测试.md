# 位置环正弦、阶跃与扫频测试

本手册用于在已经完成双端回零的丝杠轴上测试完整闭环：

```text
测试轨迹位置目标
    -> MotorControl 位置 P
    -> 距离制动速度上限
    -> 速度 PI
    -> 电流 PI
    -> PWM
```

测试轨迹只属于 `Firmware/App/screw_axis.cpp` 的 App 层。它不直接改 iq、电压或 PWM，
因此 Ozone 看到的误差、速度和电流仍是正式位置闭环的真实响应。

如果文档与源码冲突，以 `Firmware/App/screw_axis.h` 和 `screw_axis.cpp` 为准。

## 1. 前提与总入口

先完成回零。只有以下条件同时满足时，测试轨迹才会输出位置命令：

```text
g_screw_axis_watch.homed == 1
g_screw_axis_watch.travel_mm_x100 > 0
g_screw_axis_cmd.test_enable == 1
```

所有测试都通过同一个命令结构体完成：

```text
g_screw_axis_cmd
```

先写全部参数，最后写：

```text
g_screw_axis_cmd.test_enable = 1
```

停止测试：

```text
g_screw_axis_cmd.test_enable = 0
```

也可以写 `g_screw_axis_cmd.stop = 1`，它会停止电机并清掉所有位置和测试使能。

测试模式值：

| `test_mode` | 名称 | 轨迹 |
| --- | --- | --- |
| `1` | Sine | 固定频率正弦 |
| `2` | Step | 固定频率方波阶跃 |
| `3` | Sweep | 线性扫频正弦 chirp |

旧的 `sine_enable` 仍然有效，等价于 `test_mode = 1`，但新测试统一使用
`test_enable/test_mode`。

## 2. 公共单位

位置使用 `mm_x100`：

```text
100 = 1.00 mm
500 = 5.00 mm
```

频率使用 mHz：

```text
100  = 0.1 Hz
500  = 0.5 Hz
1000 = 1.0 Hz
5000 = 5.0 Hz
```

当前丝杠导程是 `0.50 mm/rev`：

```text
5000 rpm = 5000 * 0.50 / 60 = 41.7 mm/s
4200 rpm = 35.0 mm/s
```

位置模式速度上限保持 `5000 rpm`。若测试信号要求的瞬时速度超过此值，位置环会限速，
得到的是受限轨迹，不再是理想正弦或阶跃。

## 3. 正弦测试

正弦目标为：

```text
target(t) = center + amplitude * sin(2 * pi * f * t)
```

最大线速度和最大加速度为：

```text
v_peak = 2 * pi * f * amplitude
a_peak = (2 * pi * f)^2 * amplitude
```

其中 `amplitude` 使用 mm，`f` 使用 Hz。

### 3.1 约 35 mm/s 的大行程正弦

若实测行程约 `9.6 mm`，先留约 1 mm 两端余量：中心 `5.00 mm`、幅值 `4.00 mm`。
写 `1.2 Hz` 时：

```text
v_peak = 2 * pi * 1.2 * 4.0 = 30.2 mm/s
```

这是先确认整条位置环是否稳定的起点。确认无明显超调后，可升到 `1.3 Hz`：

```text
v_peak = 32.7 mm/s
```

不要直接从 `1.2 Hz` 跳到 `1.4 Hz`，因为此时约 `35.2 mm/s`，并且加速度也接近当前
制动规划能力。

```text
g_screw_axis_cmd.test_enable = 0
g_screw_axis_cmd.test_mode = 1
g_screw_axis_cmd.test_center_mm_x100 = 500
g_screw_axis_cmd.test_amplitude_mm_x100 = 400
g_screw_axis_cmd.test_frequency_mhz = 1200
g_screw_axis_cmd.position_speed_limit_rpm = 5000
g_screw_axis_cmd.position_iq_limit = 120
g_screw_axis_cmd.test_enable = 1
```

## 4. 阶跃测试

阶跃在半个周期切换一次：

```text
target(t) = center - amplitude,  前半周期
target(t) = center + amplitude,  后半周期
```

它最适合观察超调、稳定时间、速度峰值和 iq 制动方向。下面是 `1.00 mm <-> 9.00 mm`
的 8 mm 阶跃，每次到另一端后停留约 1 秒：

```text
g_screw_axis_cmd.test_enable = 0
g_screw_axis_cmd.test_mode = 2
g_screw_axis_cmd.test_center_mm_x100 = 500
g_screw_axis_cmd.test_amplitude_mm_x100 = 400
g_screw_axis_cmd.test_frequency_mhz = 500
g_screw_axis_cmd.position_speed_limit_rpm = 5000
g_screw_axis_cmd.position_iq_limit = 120
g_screw_axis_cmd.test_enable = 1
```

`500 mHz = 0.5 Hz`，一个完整来回是 2 秒，半周期就是 1 秒。

先画下面的变量：

```text
g_screw_axis_watch.test_target_mm_x100
g_screw_axis_watch.pos_mm_x100
g_screw_axis_watch.error_mm_x100
g_motor.position.brake_speed_limit_rpm
g_motor.position.speed_ref_rpm
g_motor.debug.speed_ref_active_rpm
g_motor.debug.speed_fb_rpm
g_motor.speed.iq_ref.value
g_motor.current.dq.q
```

判读顺序：

```text
test_target 已改变，但 position.speed_ref_rpm 不变
  -> 位置模式命令没有进入 MotorControl。

speed_ref_active_rpm 已下降，但 speed_fb_rpm 仍正向很高
  -> 制动转矩/速度环跟不上，不是测试信号的问题。

speed_fb_rpm 已反向，pos_mm_x100 仍持续越界
  -> 机构惯量或位置累计需要进一步检查。
```

## 5. 扫频测试

Sweep 是一次连续 chirp：

```text
f(t) = f_start + (f_end - f_start) * t / T
target(t) = center + amplitude * sin(phase(t))
phase(t) = 2 * pi * integral(f(t) dt)
```

固件用 64 位整数积分相位，并使用 ADC 采样派生的毫秒时间基。扫完 `T` 后不会跳回
起始频率，而是保持结束频率正弦；要重新扫一次，先写 `test_enable = 0`，再写回 `1`。

以下配置从 `0.1 Hz` 平滑扫到 `5 Hz`，持续 30 秒，幅值为 `1.00 mm`。在 5 Hz 时，
最大理想线速度约为 `31.4 mm/s`，低于 35 mm/s 的测试目标：

```text
g_screw_axis_cmd.test_enable = 0
g_screw_axis_cmd.test_mode = 3
g_screw_axis_cmd.test_center_mm_x100 = 500
g_screw_axis_cmd.test_amplitude_mm_x100 = 100
g_screw_axis_cmd.test_sweep_start_frequency_mhz = 100
g_screw_axis_cmd.test_sweep_end_frequency_mhz = 5000
g_screw_axis_cmd.test_sweep_duration_ms = 30000
g_screw_axis_cmd.position_speed_limit_rpm = 5000
g_screw_axis_cmd.position_iq_limit = 120
g_screw_axis_cmd.test_enable = 1
```

重点观察：

```text
g_screw_axis_watch.test_frequency_mhz
g_screw_axis_watch.test_phase
g_screw_axis_watch.test_target_mm_x100
g_screw_axis_watch.pos_mm_x100
g_motor.debug.speed_fb_rpm
g_motor.current.dq.q
```

当实际位置的幅值明显低于 `test_target_mm_x100`，或相位明显落后时，当前频率已接近或
超过位置环带宽。不要先增加位置 KP；先记录该频率，降低扫频终点或减小幅值，再分析速度
PI、iq 限幅和机械共振。

## 6. 当前默认值

| 参数 | 默认值 | 含义 |
| --- | --- | --- |
| `test_center_mm_x100` | 500 | 5.00 mm |
| `test_amplitude_mm_x100` | 100 | +/-1.00 mm |
| `test_frequency_mhz` | 500 | 0.5 Hz |
| `test_sweep_start_frequency_mhz` | 100 | 0.1 Hz |
| `test_sweep_end_frequency_mhz` | 5000 | 5 Hz |
| `test_sweep_duration_ms` | 30000 | 30 s |

这些是测试起点，不是机械保护边界。最终目标始终会按已回零的实测行程夹到
`0..travel_mm_x100`。
