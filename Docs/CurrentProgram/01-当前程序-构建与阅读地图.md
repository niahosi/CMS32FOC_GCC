# 当前程序构建与阅读地图

本文说明当前 `cms32foc` 主固件怎么阅读、CMake 怎么组织编译、参与主固件的 `.c/.cpp/.h/.S` 之间是什么关系。本文以当前工程状态为准：主固件已经进入 C/C++ 混编主线，`ScrewAxis` 和 `MotorControl` 核心调度层使用 C++ 实现，对外仍保留 C ABI；轴层 `Jog/Sweep` 测试入口已从主固件删除，MotorControl 现在提供电机级 `Position` 模式。

## 先看总图

当前主固件可以按这条线读：

```text
Reset_Handler
  -> SystemInit()
  -> main()
      -> bsp_init()
          -> BoardProfile_Init()
      -> MotorControl_Init()
      -> ScrewAxis_Init()
      -> bsp_start_adc_sync()
      -> while(1)
          -> ScrewAxis_Run()
          -> MotorControl_ApplyCommand()
          -> MotorControl_RunSlowLoop()

ADC_IRQHandler()
  -> BoardProfile_AdcIrqBegin()
  -> BoardProfile_MotorFastBegin()
  -> MotorControl_FastLoopFromAdcIrq()
      -> bsp_adc_irq()
      -> Current / Speed / Position / VF fast loop
  -> BoardProfile_MotorFastEnd()
  -> ScrewAxis_OnAdcSample()
  -> BoardProfile_AdcIrqEnd()
```

一句话：`main.c` 调度慢环和应用层，`ADC_IRQHandler()` 触发控制快环，Board 层负责真实 PWM/ADC/MA600，MotorControl 层负责 FOC 和状态机。

## CMake 管理方式

顶层 `CMakeLists.txt` 当前声明：

```cmake
project(CMS32FOC_GCC C CXX ASM)
```

也就是说当前工程已经启用 C++ 编译器。主固件当前会编译 `screw_axis.cpp` 和 `core.cpp`，但底层 Board、FOC 数学和大部分 MotorControl 快环仍保持 C。

### 公共编译选项

`cms32_project_options` 是 `INTERFACE` 库，不生成 `.a`，只传播编译参数和头文件路径。

它提供：

```text
-mcpu=cortex-m0plus
-mthumb
-ffunction-sections
-fdata-sections
-fdiagnostics-color=always
-Wall
-Wextra
```

其中 `-ffunction-sections` 和 `-fdata-sections` 配合链接参数 `--gc-sections`，让未引用函数和数据可以被链接器删除。

### 固件链接配置

`cms32_configure_firmware(target)` 定义在 `cmake/firmware_output.cmake`，给每个可烧录固件挂接：

```text
Firmware/Drivers/CMS32M6510/cms32m6510_flash.ld
-Wl,--gc-sections
-Wl,-Map=<target>.map
--specs=nano.specs
--specs=nosys.specs
```

构建后生成：

```text
<target>
<target>.hex
<target>.bin
<target>.map
```

并打印 Flash/RAM 报告。真正烧进 Flash 的是 `.text + .data`，不是带 DWARF 调试信息的 ELF 文件大小。

## 主固件源码依赖关系

### App 层

```text
Firmware/App/main.c
├── MotorControl.h
├── cms32m6510_platform.h
├── foc_bsp.h
└── screw_axis.h
```

`main.c` 只做启动和调度，不直接实现控制算法。

```text
Firmware/App/screw_axis.cpp
├── screw_axis.h
├── BoardConfig.h
├── motor_control_state.h
└── MotorControl.h
```

`screw_axis.cpp` 当前只负责：

```text
轴级命令入口 g_screw_axis_cmd
轴级观察入口 g_screw_axis_watch
双端回零状态机
记录 outer_encoder_pos 和 zero_encoder_pos
根据 outer_encoder_pos - zero_encoder_pos 更新 travel_counts
根据 encoder_pos 更新 pos_counts
把 target_mm_x100 物理位置命令换算成 MotorControl 的绝对 position_ref
回零运行、stop、fault 和回零完成瞬间向 g_mc_cmd 写速度模式命令
```

它不再包含 `Jog/Sweep` 调试逻辑，也不再包含轴层相对位置命令。真正 1 kHz
位置环现在在 MotorControl 里：`MC_MODE_POSITION` 用 `g_motor.encoder.pos`
这个绝对累计 encoder count 做反馈，目标来自 `g_mc_cmd.position_ref`。
ScrewAxis 在空闲/Ready 时不会持续覆盖 `g_mc_cmd`。如果希望用物理单位调试，
先用 `g_screw_axis_cmd.home_enable = 1` 回零；完成后固件会自动清 0。然后置
`g_screw_axis_cmd.position_enable = 1`，后续直接修改
`g_screw_axis_cmd.target_mm_x100` 就会更新 MotorControl 的 Position 目标；
例如 0.50 mm 写 50，10.00 mm 写 1000。
如果要做正弦往返测试，则置 `g_screw_axis_cmd.sine_enable = 1`，由
`sine_center_mm_x100 / sine_amplitude_mm_x100 / sine_period_ms` 自动生成位置目标。

### MotorControl 公共 API

```text
Firmware/MotorControl/Abi/MotorControl.h
└── <stdint.h>
```

这是控制层 C ABI。主循环、应用层和后续 C++ 包装层都应通过它进入控制层。

它暴露：

```text
g_mc_cmd
MotorControl_Init()
MotorControl_ApplyCommand()
MotorControl_RunSlowLoop()
MotorControl_FastLoopFromAdcIrq()
```

`g_motor.runtime`、`g_motor.command`、`g_motor.encoder`、`g_motor.position`、
`g_motor.speed`、`g_motor.current`、`g_motor.output`、`g_motor.check` 和
`g_motor.diag` 是 MotorController 对象里的真实内部状态和诊断状态。相关 C
结构体类型放在 `motor_control_state.h`，全局对象声明放在
`motor_controller.hpp`。它们不是公共控制 API，而是当前调试时给 Ozone 展开的
真实状态。

`g_motor.debug` 是 C++ 调试摘要，定义在 `debug_state.hpp/core.cpp`，用于让 Ozone
直接看到枚举状态和 rpm 观察值。

### MotorControl 内部层

```text
motor_control_state.h
├── BoardConfig.h
├── Config.h
├── MotorControl.h
├── TuneConfig.h
└── foc_math.h

motor_control_internal.h
└── motor_control_state.h
```

`motor_control_state.h` 是当前扁平状态和 Ozone 观察入口，核心是这些状态：

```text
g_motor.runtime   慢环状态机和 PWM 运行标志
g_motor.command   主循环锁存后的快环命令缓存
g_motor.position  1 kHz 位置 P 环状态
g_motor.encoder   编码器角度、位置和 raw 差分
g_motor.speed     速度估算、速度 PI 和速度输出
g_motor.current   电流采样、dq 电流、电流 PI 和电流给定
g_motor.output    dq/alpha-beta 电压和 PWM duty 输出
g_motor.check     慢环安全检查
g_motor.diag      诊断计数和拒绝详情
g_motor.debug     state/mode/fault 枚举调试入口
```

`motor_control_internal.h` 只保留 MotorControl 内部函数声明，例如 reset、快环更新、
安全态和电压输出函数。App 层如果只是观察 MotorControl 状态，应 include
`motor_control_state.h`，不要拿到全部内部函数。

Current、Encoder、Output、VF 直接操作 `g_motor.*` 里的对应状态，不再通过
旧式总状态树深层路径访问，也不再拆成多个 `g_mc_*` 状态全局。只参与观察的计数和拒绝
详情放在 `g_motor.diag`，枚举可读状态放在 `g_motor.debug`。

## FOC 调用链

### 主循环慢链路

```text
main()
  -> ScrewAxis_Run()
      -> home_update()
          -> 写 g_mc_cmd.control_mode = speed
          -> 写 g_mc_cmd.speed_ref_rpm
          -> 写 g_mc_cmd.iq_limit

  -> MotorControl_ApplyCommand(&g_mc_cmd)
      -> snapshot volatile 命令
      -> 限幅 id/iq/speed/vf/PI 参数
      -> 拆成 current/speed/position/vf 小命令缓存
      -> 模式切换时 reset PI、速度估算、VF

  -> MotorControl_RunSlowLoop()
      -> 读取慢速电流快照
      -> 检查 current_ok
      -> 检查 encoder_ok
      -> 设置 state = IDLE / CLOSED_LOOP / FAULT

  -> Ozone 直接展开 g_motor.*
      -> 不再维护慢环转发 watch
```

慢链路不做 Clarke/Park/PI/SVPWM。慢链路只负责命令和状态；观察直接看 `g_motor.*`。

### ADC 快环链路

```text
ADC_IRQHandler()
  -> MotorControl_FastLoopFromAdcIrq()
      -> bsp_adc_irq()
          -> curr_irq()
              -> 读 ADC
              -> 当前采样窗口解析
              -> 两相采样/三相重构
              -> 更新 curr_u/v/w
```

如果本次 ADC 形成有效采样：

```text
MotorControl_FastLoopFromAdcIrq()
  -> mode == Current
      -> MotorControl_CurrentRunFastLoop(speed_mode = 0)
  -> mode == Speed
      -> MotorControl_CurrentRunFastLoop(speed_mode = 1)
  -> mode == VF
      -> MotorControlVf_RunFastLoop()
```

### Current/Speed 快环

```text
MotorControl_CurrentRunFastLoop()
  -> curr_u/v/w()
  -> MotorControl_InternalCurrentOk()
  -> MotorControl_InternalUpdateEncoderAngle()
      -> bsp_update_angle_fast()
      -> ma600_update_fast()
      -> electrical_from_raw()
      -> bad angle reject / retry / hold

  -> 每 MC_SPEED_SAMPLE_DIV 次:
      -> MotorControl_InternalUpdateEncoderSpeed()
      -> speed_mode != 0:
          -> update_speed_loop()
              -> rpm/count 换算
              -> speed_ref 斜坡
              -> speed PI
              -> speed_iq_ref

  -> foc_clarke_3phase()
  -> foc_park()
  -> foc_pi_set_gains()
  -> foc_pi_update() for d/q
  -> MotorControl_InternalApplyVoltageVector()
      -> foc_limit_dq()
      -> foc_inv_park()
      -> foc_svpwm()
      -> pwm_set_duty()
      -> pwm_enable()
```

Current 模式直接使用 `g_motor.command.current.iq_ref`。Speed 模式先用速度环得到 `g_motor.speed.iq_ref.value`，再作为 q 轴电流给定。Position 模式先在 1 kHz tick 上用 `g_motor.encoder.pos` 和 `g_motor.command.position.position_ref` 计算速度目标，再复用速度环和电流环。外部 `g_mc_cmd` 只作为 Ozone/串口/App 命令入口，快环不直接读取它。

### VF 快环

```text
MotorControlVf_RunFastLoop()
  -> curr_u/v/w()
  -> MotorControl_InternalCurrentOk()
  -> 检查 open_loop_timeout
  -> update_open_loop_theta()
  -> MotorControl_InternalApplyVoltageVector(vd=0, vq=vf_voltage, theta=open_loop_theta)
  -> 定期读 MA600 角度，速度观察由 raw 差分得到
```

VF 的角度来自内部开环积分，不来自编码器。编码器在 VF 下主要用于看转子是否跟上。

## Board 层细节阅读顺序

建议不要先从原厂库读起。按当前主链路读：

```text
1. foc_bsp.c
2. foc_pwm.c
3. foc_curr.c
4. foc_ma600.c
5. 原厂 adc/epwm/gpio/pga/ssp 驱动
```

`foc_curr.c` 是 Board 层最重的文件。先抓住这条线：

```text
pwm_set_duty()
  -> curr_sync_timing()
      -> 根据 duty 选择低边采样窗口
      -> 设置 ADC 触发 tick

ADC IRQ
  -> curr_irq()
      -> 读两相
      -> 判断双点采样差值
      -> 重构第三相
      -> 相序/符号映射到逻辑 U/V/W
```
