# 高级 FOC 控制概念

本文从外部理论笔记迁入，并已按当前 GCC 工程路径和参数做同步；若后续代码与本文冲突，以当前源码为准。

本文只讲概念和当前工程关系，不要求现在实现。

当前主固件已经有：

```text
编码器 FOC
Clarke/Park/InvPark
电流 PI
速度 PI
SVPWM
低边三电阻采样
VF 开环诊断
```

当前主固件还没有：

```text
d/q 解耦前馈
反电动势前馈
弱磁
MTPA
无感观测器
PLL
滑模观测器
```

这些属于后续阶段。现在先理解它们解决什么问题。

## 1. d/q 解耦前馈

d/q 电压方程里有耦合项：

```text
vd = ... - we * Lq * iq
vq = ... + we * Ld * id + we * psi
```

电流 PI 如果不补偿这些项，就要靠反馈慢慢修正。

解耦前馈的思路：

```text
在 PI 输出之外，提前加上抵消耦合项的电压。
```

大致：

```text
vd_ff = -we * Lq * iq
vq_ff =  we * Ld * id
```

好处：

```text
高速时 d/q 电流互相干扰更小
电流环压力变小
```

前提：

```text
R/L 参数可信
速度估算可信
电角度可信
电压余量足够
```

## 2. 反电动势前馈

q 轴反电动势：

```text
vq_bemf = we * psi
```

高速时 PI 要先输出这部分电压才能维持电流。如果不做前馈，速度越高，PI 积分压力越大。

反电动势前馈：

```text
vq_ff += we * psi
```

好处：

```text
高速电流跟随更好
速度变化时 vq 不必全靠积分建立
```

风险：

```text
Ke/psi 不准会引入错误电压
速度反馈噪声会进入电压输出
电压限幅仍然存在
```

## 3. 弱磁

当速度升高，反电动势接近母线可用电压时：

```text
vq 饱和
iq 跟不上
速度上不去
```

弱磁的思路是给负 `id`，削弱等效磁链：

```text
id < 0
we * psi_effective 下降
高速电压余量增加
```

代价：

```text
同样 iq 下转矩可能下降
电流总幅值增加
铜耗增加
控制复杂度提高
```

当前阶段不要急着实现弱磁。先确认基础电流环、角度和电压余量判断可靠。

## 4. MTPA

MTPA 是 Maximum Torque Per Ampere，意思是同样电流产生最大转矩。

对表贴 PMSM：

```text
Ld ~= Lq
id = 0 通常接近最优
```

对内嵌式 PMSM：

```text
Ld != Lq
合适的负 id 可以利用磁阻转矩
```

当前教学阶段先用：

```text
id_ref = 0
iq 控转矩
```

等确认电机类型和 Ld/Lq 后，再考虑 MTPA。

## 5. 无感观测器

无感控制不使用编码器直接给角度，而是从电压、电流估计转子位置。

常见信息来源：

```text
反电动势
磁链观测
电流响应
高频注入
```

低速难点：

```text
反电动势小
信噪比低
参数误差影响大
```

当前工程已有 MA600 编码器，所以优先把有感 FOC 做稳。无感观测器可以作为后续学习主题。

## 6. PLL

PLL 用于从估计角度/反电动势里提取连续角度和速度。

概念上：

```text
相位误差
  -> PI/滤波
  -> 估计速度
  -> 积分得到估计角度
```

PLL 的调参问题和速度环类似：

```text
带宽高: 跟踪快但噪声大
带宽低: 平滑但延迟大
```

当前编码器速度估算用 raw 差分和一阶低通，还不是 PLL。

## 7. 什么时候适合引入高级控制

先满足：

```text
Current 模式小 iq_ref 稳定
id 接近 0
iq 方向正确
v_limited 判断清楚
速度反馈方向正确
Speed 模式低速可控
采样噪声可接受
```

再考虑：

```text
高速 v_limited 明显:
  反电动势前馈或弱磁。

高速 d/q 互相影响明显:
  d/q 解耦前馈。

追求效率:
  MTPA。

希望去掉编码器:
  无感观测器。
```

不要在基础相序、零位、采样还不稳时引入高级算法。高级算法会把问题叠得更深。
