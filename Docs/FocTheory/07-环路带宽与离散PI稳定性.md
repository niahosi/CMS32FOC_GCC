# 环路带宽与离散 PI 稳定性

本文从外部理论笔记迁入，并已按当前 GCC 工程路径和参数做同步；若后续代码与本文冲突，以当前源码为准。

本文讲当前工程里电流环、速度环为什么不能只按“参数越大响应越快”理解。

FOC 是离散控制系统：

```text
ADC 采样
  -> 计算
  -> 更新 PWM
  -> 下一拍电流才变化
```

每一步都有采样周期、计算延迟、量化、限幅和噪声。PI 参数必须和这些约束一起看。

## 1. 当前控制频率

当前配置：

```text
PWM_FREQ_HZ = 20000
CTRL_FAST_LOOP_DIV = 1
MC_CURRENT_HZ = 20000 Hz
CTRL_SPD_EST_HZ = 1000 Hz
MC_SPEED_SAMPLE_DIV = 20
```

所以：

```text
电流环周期 Ts_current = 50 us
速度环周期 Ts_speed = 1 ms
```

这决定了两个原则：

```text
电流环是内环，必须更快。
速度环是外环，必须更慢。
```

如果速度环太激进，它会快速改变 `iq_ref`，但电流环和电机来不及跟上，
表现就是速度抖动、`speed_iq_cmd` 打满、`vq` 饱和。

## 2. 采样频率不等于可用带宽

20 kHz 电流环不代表可以把电流环带宽设到 10 kHz。

原因：

```text
ADC 采样不是连续的。
PWM 更新有一拍延迟。
电流采样有窗口限制。
Park 使用的角度也有采样时刻。
整数 PI 有量化。
SVPWM 有限幅。
```

保守起步可以先把电流环目标带宽放在：

```text
1 kHz 附近
```

再根据阶跃响应逐步调。对于当前低电感电机，理论电气时间常数可能很短，
但实际系统还受采样噪声、死区、角度误差和电压饱和限制。

## 3. 当前 PI 的离散形式

当前 PI：

```text
error = ref - feedback
integral += error
output = (kp * error + ki * integral) >> shift
```

电流环：

```text
CTRL_CUR_KP = 4
CTRL_CUR_KI = 1
CTRL_CUR_PI_SHIFT = 3
```

速度环：

```text
CTRL_SPD_KP = 32
CTRL_SPD_KI = 3
CTRL_SPD_ERR_SHIFT = 10
```

注意积分项的含义：

```text
积分不是按秒累加，而是每个控制周期累加一次 error。
```

所以同一个 `KI`，在 20 kHz 电流环和 1 kHz 速度环里完全不是同一个物理强度。

## 4. KP 太大是什么现象

KP 提高后，误差一出现就会立刻给更大的输出。

好处：

```text
响应更快
误差更快被压小
```

代价：

```text
放大采样噪声
更容易 duty 抖动
更容易触发电压限幅
角度或相序小错误会被放大
```

看数据：

```text
id/iq 高频抖动
vd/vq 高频跳动
duty_u/v/w 抖动
v_limited 偶发或持续为 1
电机有高频声音
```

处理：

```text
先确认采样和角度没错。
如果采样可信但高频抖，降低 KP。
如果低速小电流都抖，不要先加 KI。
```

## 5. KI 太大是什么现象

KI 用来消除稳态误差。

好处：

```text
长期误差会被积分推掉。
```

代价：

```text
容易积分饱和。
容易过冲。
输出碰到电压限幅后，积分会让恢复变慢。
零速附近可能保持一个不该有的输出。
```

当前 `foc_pi_update()` 有简单抗积分饱和：输出已经饱和且误差还在把输出往更饱和方向推时，
积分不会继续累加。

但这不是万能的。如果目标电流超过物理可实现电流，PI 仍然会长期贴着电压限幅。

看数据：

```text
iq 慢慢爬过目标
vd/vq 长时间贴限幅
v_limited 长期为 1
降低目标后恢复慢
零速附近 speed_pi_integral 不合理
```

处理：

```text
确认不是电压余量不足。
确认不是角度/相序错误。
若不饱和但稳态误差大，再增 KI。
若饱和或过冲明显，降 KI。
```

## 6. 限幅改变环路性质

理想 PI 推导通常假设输出不限幅。但当前工程有多层限幅：

```text
iq_limit
CTRL_CUR_REF_LIMIT
current_v_limit
CTRL_CUR_V_LIMIT
PWM_DUTY_MIN / PWM_DUTY_MAX
foc_limit_dq()
```

一旦限幅生效，系统就不是原来的线性 PI 闭环。

关键字段：

```text
v_limited
duty_u/duty_v/duty_w
speed_iq_cmd
speed_pi_integral
```

判断：

```text
v_limited = 0:
  可以讨论 PI 参数是否合适。

v_limited = 1:
  先讨论电压余量、相位、负载和目标是否合理。
```

## 7. 速度环为什么要慢

速度环控制对象不是电流，而是机械系统：

```text
J * d(w)/dt = Te - Tload - B*w
```

速度环输出：

```text
speed_iq_cmd -> iq_ref
```

然后电流环再把 `iq_ref` 变成实际 `iq`。

如果速度环带宽接近或超过电流环，外环会命令内环做不到的变化：

```text
speed_err_rpm 大
speed_iq_cmd 快速打满
iq 跟不上 iq_ref
vq 饱和
速度抖动
```

所以速度环调试顺序：

```text
1. Current 模式确认 iq 能稳定跟随 iq_ref。
2. 限制 iq_limit。
3. 用小 speed_ref_rpm 测速度方向和响应。
4. 再逐步调 speed_kp。
5. 最后加 speed_ki 消除稳态误差。
```

## 8. 速度反馈滤波和延迟

当前速度估算：

```text
speed_sample = delta * CTRL_SPD_EST_HZ * MOT_SENSOR_DIR
speed_fb_diff += (speed_sample - speed_fb_diff) >> CTRL_SPD_FILTER_SHIFT
```

当前：

```text
CTRL_SPD_FILTER_SHIFT = 2
```

这相当于一阶低通：

```text
new = old + (sample - old) / 4
```

滤波越强：

```text
速度反馈更平滑
延迟更大
速度环更容易滞后
```

滤波越弱：

```text
响应更快
速度噪声更明显
速度 PI 更容易抖
```

调速度环时要同时看：

```text
speed_fb_rpm
speed_err_rpm
speed_iq_cmd
speed_pi_integral
encoder_delta
speed_reject_count
```

## 9. 推荐调参判断

电流环：

| 现象 | 判断 | 动作 |
| --- | --- | --- |
| `iq` 慢，`v_limited=0` | 可能 PI 偏弱 | 增 KP 或 KI |
| `iq` 抖，`vd/vq` 抖 | KP 可能偏大或采样噪声 | 先查采样，再降 KP |
| 稳态误差大，未饱和 | KI 偏弱 | 增 KI |
| 过冲大，恢复慢 | KI 偏强或饱和 | 降 KI，查限幅 |
| `v_limited=1` | 电压不够或相位错误 | 先不调 PI |

速度环：

| 现象 | 判断 | 动作 |
| --- | --- | --- |
| 速度慢慢到目标 | 斜坡/PI 保守 | 可适度增 KP |
| 速度来回摆 | KP 太大、滤波延迟、机械共振 | 降 KP 或查机械 |
| 长期有误差，iq 未限幅 | KI 偏弱 | 增 KI |
| `speed_iq_cmd` 打满 | 外环已经尽力 | 查 iq、电压、负载 |
| 低速抖 | 死区、速度量化、积分保持 | 查死区和 KI |

## 10. 最小实验建议

电流环实验：

```text
Current 模式
id_ref = 0
iq_ref = 20, 40, 60 count 分级
iq_limit 略高于 iq_ref
观察 id/iq/vd/vq/v_limited/duty
```

速度环实验：

```text
Speed 模式
speed_ref_rpm = 50, 100, 200 分级
iq_limit 先小
观察 speed_ref_active_rpm/speed_fb_rpm/speed_iq_cmd/iq/v_limited
```

每次只改一个参数。否则数据无法说明问题来自哪里。
