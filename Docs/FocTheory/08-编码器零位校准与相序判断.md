# 编码器零位校准与相序判断

本文从外部理论笔记迁入，并已按当前 GCC 工程路径和参数做同步；若后续代码与本文冲突，以当前源码为准。

本文讲 FOC 里最常见、也最容易误判的问题：电角度零位、方向、PWM 相序、电流相序。

这些问题的表现很像 PI 参数错误：

```text
电机抖
iq 跟不上
id 很大
vd/vq 很快饱和
duty 贴边
```

但根因不是 PI，而是 FOC 坐标系没有对准真实电机。

## 1. 当前工程的角度公式

当前编码器映射：

```text
encoder_elec = MOT_ELEC_ZERO + elec_zero_trim + raw
```

如果 `MOT_SENSOR_DIR < 0`，则 raw 方向反过来。

当前配置：

```text
MOT_POLE_PAIRS = 4
MOT_SENSOR_CPR = 65536
MOT_SENSOR_DIR = 1
MOT_ELEC_ZERO = -13478
```

当前磁环极对数和电机极对数一致，MA600 raw 已经是电角度标尺，所以这里没有额外的传感器角度倍数。注意这不代表机械圈换算里不需要 `MOT_POLE_PAIRS`；一机械圈仍然对应 4 个电角周期。

运行时可以临时调：

```text
elec_zero_trim
voltage_theta_offset
```

区别：

```text
elec_zero_trim:
  改编码器电角度零位，影响 encoder_elec。

voltage_theta_offset:
  只给电流环使用的 theta 加偏置，不改变 encoder_elec 观察值。
```

## 2. 为什么零位错会把 id/iq 搞混

Park 变换依赖电角度：

```text
d =  alpha * cos(theta) + beta * sin(theta)
q = -alpha * sin(theta) + beta * cos(theta)
```

如果 `theta` 比真实电角度偏了 `delta`，那么测出来的 `id/iq` 就是旋转错位后的结果。

直观理解：

```text
零位正确:
  q 轴电流主要显示在 iq。

零位偏 90 电角度:
  q 轴电流会大幅跑到 id。

零位偏 180 电角度:
  iq 方向可能反。
```

所以判断零位时要看：

```text
id_ref = 0
小 iq_ref
id 是否接近 0
iq 是否跟随 iq_ref
vd/vq 是否不过早饱和
```

## 3. 方向错和零位错的区别

方向错通常表现为：

```text
手转时 speed_fb_rpm 符号和实际方向相反。
给正 iq_ref，电机转向和期望相反。
闭环一动起来角度越用越错，容易抖动或 fault。
```

零位错通常表现为：

```text
低速或堵转时也能看到 id/iq 混叠。
调整 elec_zero_trim 后 id/iq 分布明显变化。
速度方向不一定反，但转矩效率差。
```

先判断方向，再判断零位。方向错时调零位只能局部改善，不能根治。

## 4. PWM 相序、电流相序、电流符号

当前有三组方向/相序：

```text
MOT_PWM_PHASE_MAP:
  控制算法 U/V/W 到实际 PWM U/V/W。

MOT_CURR_PHASE_MAP:
  实际采样 U/V/W 到控制算法 U/V/W。

MOT_CURR_SIGN:
  电流整体符号。
```

它们必须和编码器方向共同一致。

典型错误：

```text
PWM 相序错:
  输出的旋转电压和算法认为的相序不一致。

电流相序错:
  采样电流被送进错误的 U/V/W。

电流符号错:
  PI 以为在减小误差，实际在增大误差。
```

这些问题都可能表现为：

```text
iq 方向反
id 很大
vd/vq 快速变大
v_limited = 1
电机抖或反拖
```

## 5. VF 模式用于分层排查

VF 模式不依赖编码器角度闭环。它用开环角输出旋转电压：

```text
vd = 0
vq = vf_voltage
theta = open_loop_theta
```

所以 VF 可以先验证：

```text
PWM 是否能输出
SVPWM duty 是否随 open_loop_theta 变化
功率级是否能推动电机
电流采样是否有响应
编码器 speed_fb_rpm 是否跟实际方向一致
```

判断：

```text
VF 模式 duty 变、电机有反应、编码器速度方向正确:
  底层输出和编码器大体可用，再查闭环零位和电流方向。

VF 模式 duty 变但电机无反应:
  查功率级、驱动使能、接线、PWM 相序。

VF 模式电机动但 speed_fb_rpm 方向反:
  先查 MOT_SENSOR_DIR。
```

## 6. Current 模式小电流测试

基础测试：

```text
control_mode = Current
id_ref = 0
iq_ref = 20 或 40 count
iq_limit 略高于 iq_ref
current_v_limit 保持默认
```

看：

```text
id_ref
iq_ref
id
iq
vd
vq
v_limited
duty_u/v/w
encoder_elec
```

正常倾向：

```text
iq 跟随 iq_ref
id 接近 0
vd/vq 不应立刻打满
v_limited = 0
duty 不贴边
```

异常和判断：

| 现象 | 优先怀疑 |
| --- | --- |
| `iq` 方向反 | 电流符号、传感器方向、相序 |
| `id` 很大 | 零位偏、Park 角度错 |
| `vq` 很大但 `iq` 不动 | 功率级、相序、角度 |
| `v_limited=1` | 相位错或目标/负载过大 |
| 电机抖但不连续转 | 零位、相序、采样噪声 |

## 7. 用角度偏置判断零位方向

可以临时扫 `voltage_theta_offset` 或 `elec_zero_trim`。

建议方法：

```text
1. 固定小 iq_ref。
2. 每次只改一个小步，例如 512 或 1024 个 16-bit 角度 count。
3. 观察 id 是否变小、iq 是否更接近 iq_ref。
4. 观察 vd/vq 和 v_limited 是否变好。
```

16-bit 周期角中：

```text
65536 count = 360 电角度
182 count ~= 1 电角度
1024 count ~= 5.6 电角度
```

判断：

```text
某个方向调 offset 后 id 变小、iq 变合理、vq 下降:
  零位应该往这个方向修。

两个方向都变差:
  可能不是零位问题，查相序/电流符号/采样。

需要接近 90 度的大偏置才好:
  可能相序或 d/q 方向基础定义有错。
```

## 8. 对齐理论

对齐的思想是：给固定 d 轴电压，把转子磁场拉到一个已知电角度。

当前相关参数：

```text
MOT_ALIGN_VD
MOT_ALIGN_THETA
MOT_ALIGN_TICKS
MOT_ALIGN_SCAN_VD
MOT_ALIGN_SCAN_FAST_SPEED
MOT_ALIGN_SCAN_SLOW_SPEED
```

主固件当前不直接支持冻结的对齐诊断模式，但这些参数保留了思路：

```text
用 d 轴电压建立已知磁场方向。
记录此时编码器 raw。
把目标电角度和 raw 的差作为零位。
```

实际对齐会受：

```text
摩擦
齿槽转矩
负载
电压太小拉不动
电压太大过热或冲击
```

影响。因此正反拖动扫描可以减小单方向摩擦和齿槽带来的偏差。


## 10. 推荐排查顺序

```text
1. VF 模式确认 PWM、功率级、电流采样、编码器速度方向。
2. 手转确认 encoder_raw 连续，speed_fb_rpm 符号正确。
3. Current 模式小 iq_ref，观察 id/iq。
4. 用 voltage_theta_offset 小步扫描零位方向。
5. 若 offset 怎么调都不对，查 PWM 相序和电流相序。
6. 最后固化 MOT_SENSOR_DIR、MOT_ELEC_ZERO、相序和电流符号。
```

不要在方向、相序、零位未确认前调大电流 PI 或速度 PI。
