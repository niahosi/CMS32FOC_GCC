# 保护逻辑与安全状态理论

本文从外部理论笔记迁入，并已按当前 GCC 工程路径和参数做同步；若后续代码与本文冲突，以当前源码为准。

本文讲当前工程里的保护思想。

FOC 控制器不是只追求转起来。电机控制首先要能在数据不可信或输出危险时停下来。

当前主线状态：

```text
MC_STATE_IDLE = 0
MC_STATE_CLOSED_LOOP = 3
MC_STATE_FAULT = 4
```

当前故障：

```text
MC_FAULT_NONE = 0
MC_FAULT_UNSUPPORTED_MODE = 1
MC_FAULT_CURRENT = 2
MC_FAULT_OPEN_LOOP_TIMEOUT = 3
MC_FAULT_ENCODER = 4
```

watch 里看：

```text
state
fault_reason
safe_state_count
check.current_ok
check.ma600_ok
check.pwm_off_safe
check.ready_closed_loop
pwm_safe
pwm_running
```

## 1. 为什么保护不能只靠 PI

PI 只知道：

```text
ref - feedback
```

如果 feedback 错了，PI 会认真地朝错误方向输出。比如：

```text
电流采样错:
  PI 可能继续加电压。

电角度错:
  PI 可能把 q 轴电流推到错误方向。

编码器坏角:
  Park/InvPark 使用错误 theta。

功率级异常:
  duty 输出和实际相电压不一致。
```

所以保护层必须独立判断：

```text
电流是否可信
编码器是否可信
模式是否支持
PWM 关闭时是否安全
开环是否超时
```

## 2. 慢环 ready 检查

当前慢环会刷新：

```text
current_ok
pwm_off_safe
ma600_ok
ready_closed_loop
```

Current/Speed 闭环要求：

```text
current_ok = 1
encoder 可用或还未初始化首帧
```

VF 开环不依赖闭环电角度，但仍要求：

```text
current_ok = 1
```

如果 `ready_closed_loop = 0`，不要先调 PI。先看：

```text
check.current_ok
check.ma600_ok
fault_reason
```

## 3. 电流保护

当前 `MotorControl_InternalCurrentOk()` 检查三相电流和 KCL 和，实际阈值由配置控制：

```text
MOT_CHECK_CURR_CNT_LIMIT
MOT_CHECK_SUM_CNT_LIMIT
```

当前 `TuneConfig.h` 里这些硬检查放得很宽，但采样层还有：

```text
CS_SAMPLE_ABS_HARD_LIMIT_CNT = 8192
```

电流保护要防两类问题：

```text
真实过流:
  电机/功率级可能损坏。

采样异常:
  PI 看到错误电流会输出错误电压。
```

watch 判断：

```text
fault_reason = MC_FAULT_CURRENT
check.current_ok = 0
i_sum 异常
iu/iv/iw 某相异常
safe_state_count 增长
```

处理：

```text
立即降低目标或关闭 enable。
检查 PWM 是否安全关闭。
确认零漂和采样窗口。
不要直接放宽阈值来让电机继续跑。
```

## 4. 编码器保护

闭环 FOC 需要可信电角度。

当前生产 C++ 路径把编码器保护分成两层：

```text
MOT_ANGLE_MAX_AGE
MOT_ENCODER_MAX_STEP_RAW
CTRL_SPD_DIFF_SPIKE_RPM
encoder_reject_count
encoder_hold_count
encoder_age
encoder_ok
```

保护思路：

```text
单次读角失败:
  保持上一角度。

单次 raw 跳变过大:
  不写入 encoder_pos，不更新电角度，保持上一角度。

连续读角失败:
  encoder_age 增长，超过窗口后不能继续闭环。
```

watch 判断：

```text
fault_reason = MC_FAULT_ENCODER
encoder_ok = 0
encoder_age 增长
encoder_reject_count 增长
encoder_hold_count 增长
```

处理：

```text
先查 MA600 供电、磁体、SPI 干扰。
再查 PWM/电流扰动是否污染 SPI。
不要把坏角阈值无限放宽。
```

## 5. VF 开环超时

VF 模式绕过闭环电角度，适合诊断，但不适合长期运行。

当前保护：

```text
open_loop_timeout_ms
MC_FAULT_OPEN_LOOP_TIMEOUT
```

意义：

```text
避免开环电压长时间输出导致发热。
避免调试忘记关闭。
```

watch：

```text
open_loop_ticks
open_loop_reset_count
fault_reason
vf_voltage
```

如果 VF 超时 fault，是正常保护行为，不是电流环错误。

## 6. 安全态

进入安全态时会：

```text
pwm_off()
pwm_output = 0
voltage_dq = 0
voltage_ab = 0
duty = 800/800/800
speed_iq_ref = 0
reset speed PI
safe_state_count++
```

watch 判断：

```text
safe_state_count 增长
pwm_running = 0
pwm_safe = 1
duty_u/v/w 接近 800
vd/vq = 0
```

如果 fault 后 `pwm_running` 仍为 1 或 `pwm_safe = 0`，这是比 PI 调参更高优先级的问题。

## 7. 故障恢复原则

建议原则：

```text
1. 先让 enable = 0。
2. 确认 pwm_safe = 1。
3. 记录 fault_reason 和关键计数。
4. 修复根因。
5. 重新使能前降低 iq_limit / vf_voltage / speed_ref。
```

不要用反复 enable/disable 来“撞运气”。如果 `safe_state_count` 持续增长，说明保护层在不断阻止一个真实问题。
