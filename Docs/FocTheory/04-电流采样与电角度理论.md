# 电流采样与电角度理论

本文从外部理论笔记迁入，并已按当前 GCC 工程路径和参数做同步；若后续代码与本文冲突，以当前源码为准。

本文讲 FOC 闭环能不能成立的两个前提：

```text
1. 电流采样必须可信。
2. 电角度必须可信。
```

电流 PI、速度 PI、SVPWM 都是在这两个前提上工作。如果电流或角度错了，
后面的控制器会把错误数据当真，表现出来就像 PI 参数、负载或电压出了问题。

当前相关代码：

```text
Firmware/Board/Src/foc_curr.c
Firmware/MotorControl/Core/encoder.cpp
Firmware/MotorControl/Core/current.cpp
Firmware/Board/Config/BoardConfig.h
Firmware/Board/Config/TuneConfig.h
```

## 1. 为什么电流采样是 FOC 的地基

FOC 的电流环比较的是：

```text
id_ref - id
iq_ref - iq
```

其中 `id/iq` 来自：

```text
iu/iv/iw -> Clarke -> Park
```

所以如果 `iu/iv/iw` 本身不可信，`id/iq` 就不可信，PI 输出的 `vd/vq`
也会跟着错。

当前 Ozone 里先看：

```text
g_motor.current.phase.u
g_motor.current.phase.v
g_motor.current.phase.w
g_motor.current.dq.d
g_motor.current.dq.q
g_motor.current.iq_ref_active.value
```

最基本判断：

```text
PWM 关闭或小电流时，iu/iv/iw 不应有很大偏置。
i_sum 不应长期很大。
给小 iq_ref 时，iq 应该比 id 更明显响应。
```

## 2. 三相电流和两相重构

三相星型无中线电机理想满足：

```text
Iu + Iv + Iw = 0
```

所以只采任意两相也能重构第三相：

```text
采 U/V: W = -U - V
采 U/W: V = -U - W
采 V/W: U = -V - W
```

当前工程的低边三电阻采样不保证每个 PWM 周期都采三相。它会根据 duty
排序后的 T1/T2/T3 窗口选择可采样 pair。

关键配置：

```text
CS_PAIR_UV / CS_PAIR_UW / CS_PAIR_VW
CS_USE_2PHASE_IN_ALL_WINDOW = 1
CS_MULTI_EN = 0
CS_OPEN_SETTLE_TICK = PWM_DEADTIME_TICKS + 4
CS_TAIL_MARGIN_TICK = 60
```

这里的重点不是“采得越多越好”，而是“在开关噪声较少的窗口采到可信样本”。

## 3. 低边采样窗口

低边采样要求某相下桥臂导通，采样电阻上才有可靠电流信息。中心对齐 PWM
下，每相 duty 不同，低边导通窗口也不同。

当前程序按 duty 从大到小排序：

```text
max duty
mid duty
min duty
```

然后得到三个窗口量：

```text
t1 = min duty
t2 = mid duty - min duty
t3 = max duty - mid duty
```

程序根据这些窗口选择：

```text
t1 足够: 三相都在有效窗口，但当前仍默认只采两相
t2 足够: 采两个有效相，用 KCL 重构第三相
t3 足够: 只部分可信，缺失相用滤波保持
都不够: 保持上一采样配置
```

窗口要避开两个区域：

```text
刚开通后的死区和恢复区
窗口尾部接近切换边沿的噪声区
```

对应配置：

```text
CS_OPEN_SETTLE_TICK
CS_TAIL_MARGIN_TICK
```

## 4. 单点采样与备用双点参数

当前 `CS_MULTI_EN = 0`，每个 PWM 周期只在选中的稳定低边窗口中心触发一次 ADC。
仍然按 T1/T2/T3 选择 U/V、U/W 或 V/W 两相，并按 KCL 重构第三相。

采样点由以下余量确定：

```text
CS_OPEN_SETTLE_TICK: 下管打开后等待死区和恢复过程结束。
CS_TAIL_MARGIN_TICK: 与窗口尾部切换边沿保持距离。
```

`CS_MULTI_DELTA_TICK` 和 `CS_MULTI_SPREAD_LIMIT_CNT` 保留为以后恢复双点 A/B
采样时的配置；单点模式不触发 CMP1、不等待第二个样本，也不做平均或 A/B 差值判定。

如果你看到电流偶发尖峰，先考虑：

```text
采样点贴近边沿
duty 进入高调制区
死区/恢复时间不够
ADC 顺序扫描时差
功率地噪声
```

## 5. raw current 和 logic current

当前工程区分两类电流：

```text
physical/raw current:
  ADC/PGA 通道扣零漂后的物理相电流。

logic current:
  经过相序映射和符号处理后送进 FOC 的 U/V/W。
```

FOC 用的是 logic current。映射由这些宏决定：

```text
MOT_CURR_PHASE_MAP
MOT_CURR_SIGN
```

如果硬件采样看起来正常，但给 `iq_ref` 后 `iq` 方向反，可能不是 ADC 坏了，
而是 logic current 的方向或相序与 PWM/编码器方向不一致。

## 6. 电角度是什么

FOC 需要的是电角度，不是机械角度。

关系是：

```text
electrical_angle = mechanical_angle * motor_pole_pairs
```

当前工程用 MA600 raw 角度：

```text
encoder_raw: 0..65535
encoder_elec: 0..65535 周期电角度
```

当前配置：

```text
MOT_POLE_PAIRS = 4
MOT_SENSOR_CPR = 65536
MOT_SENSOR_DIR = 1
MOT_ELEC_ZERO = -13478
```

当前磁环极对数和电机极对数一致，MA600 raw 已经是 16-bit 电角度标尺；程序不再保留独立的“传感器极对数/传感器到电角度倍数”配置。

所以当前 raw 到电角度约等于：

```text
encoder_elec = MOT_ELEC_ZERO + elec_zero_trim + encoder_raw
```

`uint16_t` 自动回绕，所以结果仍是一个 16-bit 周期角。

## 7. 零位和方向为什么关键

Park 变换依赖 `theta`。如果 `theta` 错，FOC 的坐标轴就错。

典型错误：

```text
零位错 90 电角度:
  d/q 轴互相混叠。

方向反:
  正 iq_ref 可能产生反向转矩。

PWM 相序和电流相序不一致:
  PI 以为在修正 q 轴，实际推向错误相位。
```

当前可以临时调：

```text
elec_zero_trim
voltage_theta_offset
```

但这两个不是最终替代品。最终仍应确认：

```text
MOT_ELEC_ZERO
MOT_SENSOR_DIR
MOT_PWM_PHASE_MAP
MOT_CURR_PHASE_MAP
MOT_CURR_SIGN
```

## 8. 编码器读角失败和保持

当前生产 C++ 路径不会用很小的 raw 跳变阈值去卡高速测试，但会在 20 kHz
角度接受点挡掉明显不可能的 raw 跳变。有效 raw 先经过位置累计门限，再更新
`encoder.raw`、`encoder.elec` 和 `encoder.pos`。

```text
MOT_ANGLE_MAX_AGE = 20
MOT_ENCODER_MAX_STEP_RAW = 2048
```

如果本次 SPI/Board 层读角失败，会保持上一角度；连续失败超过窗口后进入
`MC_FAULT_ENCODER`。如果本次 raw 跳变超过按快环频率和速度上限推导出的范围，
本次 raw 也会被拒绝，`encoder_reject_count` 增长。

watch 字段：

```text
encoder_ok
encoder_age
encoder_raw_step
encoder_reject_count
encoder_reject_step
encoder_hold_count
```

判断：

```text
encoder_hold_count 增长:
  读角失败后保持上一角度。

encoder_reject_count 增长:
  raw 单拍跳变不可信，位置累计本次没有写入。

encoder_age 接近或超过 20:
  闭环角度已经过旧，可能进入编码器 fault。
```

## 9. 从数据判断采样还是角度问题

### 9.1 更像电流采样问题

看见这些现象，优先查采样：

```text
i_sum 长期很大
iu/iv/iw 某一相明显偏置
PWM 关闭时电流 count 不接近零
小电流下 id/iq 噪声大
duty 变化后电流没有合理响应
```

下一步：

```text
确认零漂校准是否执行
确认采样窗口是否足够
降低 vf_voltage 或 iq_ref
检查采样电阻/PGA/ADC 通道
检查 MOT_CURR_PHASE_MAP 和 MOT_CURR_SIGN
```

### 9.2 更像电角度问题

看见这些现象，优先查角度：

```text
encoder_reject_count 或 encoder_hold_count 增长
speed_fb_rpm 符号和实际方向相反
给正 iq_ref 后 iq 方向反
给 iq_ref 后 id 大幅变化
vd/vq 很快变大并饱和
电机只抖不转
```

下一步：

```text
先用 VF 模式验证 PWM、电流采样、编码器观察是否都有响应。
再用很小 iq_ref 测正负方向。
调整 elec_zero_trim 或 voltage_theta_offset 观察 id/iq 是否变合理。
最终固化 MOT_SENSOR_DIR 和 MOT_ELEC_ZERO。
```

## 10. 学习时的最小实验顺序

建议按这个顺序验证：

```text
1. PWM 关闭，观察 iu/iv/iw/i_sum 是否接近零。
2. VF 小电压开环，观察 duty 是否变化、电流是否有响应。
3. 手动转电机，观察 encoder_raw 是否连续、speed_fb_rpm 符号是否合理。
4. Current 模式小 iq_ref，观察 iq 是否跟随、id 是否接近 0。
5. 看 vd/vq 和 v_limited，确认不是一开始就电压饱和。
6. 最后再进入 Speed 模式。
```

这套顺序的目的很明确：先证明测量可信，再证明角度可信，再证明电流环可信，
最后才让速度环参与。
