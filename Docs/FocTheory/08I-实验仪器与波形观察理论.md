# 实验仪器与波形观察理论

本文从外部理论笔记迁入，并已按当前 GCC 工程路径和参数做同步；若后续代码与本文冲突，以当前源码为准。

本文讲 FOC 调试时各种观察手段能看什么、不能看什么。

当前工程主要观察入口：

```text
g_motor.runtime
g_motor.encoder
g_motor.speed
g_motor.current
g_motor.output
g_motor.check
g_mc_cmd
g_screw_axis_watch
g_screw_axis_cmd
```

Ozone 直接看 `g_mc_*` 很有用，但它不是 20 kHz 电流环波形仪。

## 1. Ozone watch 适合看什么

适合看：

```text
状态机
故障原因
计数器是否增长
慢速趋势
命令是否进入内部
速度环 1 kHz 级别趋势
编码器 reject/hold 计数
v_limited 是否发生
```

典型字段：

```text
state
fault_reason
slow_loop_count
adc_sample_count
fast_loop_count
encoder_ok
encoder_reject_count
iu_cnt/iv_cnt/iw_cnt
id/iq
vd/vq
v_limited
duty_u/v/w
speed_fb_rpm
speed_iq_cmd
```

## 2. Ozone watch 不适合看什么

不适合看：

```text
20 kHz 电流波形细节
PWM 边沿
ADC 采样瞬间毛刺
死区引起的细小相电压误差
高频电流纹波
单个控制周期内的超调
```

原因：

```text
watch 刷新率远低于控制频率
展开大结构会增加调试访问负担
读取时刻和控制周期不同步
看到的是抽样后的慢画面
```

所以 watch 只能回答：

```text
趋势是什么？
状态是否变化？
是否饱和？
是否 fault？
```

不能替代示波器判断高频波形。

## 3. 推荐 watch 分组

不要一次展开整个结构。分组看。

状态组：

```text
g_motor.runtime.state
g_motor.runtime.mode
g_motor.runtime.fault
g_motor.check.current_ok
g_motor.check.ma600_ok
g_motor.diag.slow_loop_count
g_motor.diag.fast_loop_count
g_motor.diag.safe_state_count
```

电流组：

```text
g_motor.current.phase.u
g_motor.current.phase.v
g_motor.current.phase.w
g_motor.current.dq.d
g_motor.current.dq.q
g_motor.current.id_ref_active.value
g_motor.current.iq_ref_active.value
```

电压/PWM 组：

```text
g_motor.output.voltage_dq.d
g_motor.output.voltage_dq.q
g_motor.output.voltage_limited
g_motor.output.duty.u
g_motor.output.duty.v
g_motor.output.duty.w
```

速度组：

```text
g_motor.debug.speed_ref_cmd_rpm
g_motor.debug.speed_ref_active_rpm
g_motor.debug.speed_fb_rpm
g_motor.speed.iq_ref.value
g_motor.speed.pi.integral
```

## 4. 示波器适合看什么

示波器适合看：

```text
PWM 相输出
半桥开关波形
母线电压跌落
采样电阻两端波形
相电流纹波
ADC 触发相对 PWM 的时刻
死区和边沿振铃
```

如果你怀疑：

```text
采样窗口不干净
死区影响
功率级没有真正输出
母线掉压
高频振铃
```

应该上示波器，而不是继续看 watch。

## 5. 电流探头和采样电阻

测电流有两种思路：

```text
电流探头:
  直接看相线电流或母线电流，侵入小。

采样电阻:
  看低边采样瞬间电压，能验证 ADC 采样窗口。
```

低边采样要注意：

```text
只有下管导通窗口内才有对应相电流信息。
示波器地线连接要小心，避免短路或引入地回路。
差分探头更安全。
```

## 6. DAC / GPIO 调试输出

如果芯片资源允许，可以把内部变量输出到 DAC 或 GPIO。

适合输出：

```text
ADC IRQ 进入标记
采样触发点
某个限幅标志
简化后的 id/iq
控制周期脉冲
```

优点：

```text
能和示波器 PWM 波形对齐
能看实时周期
不依赖调试器 watch 刷新
```

缺点：

```text
需要占用引脚/外设
需要注意不要增加快环太多负担
```

## 7. RTT/SWO/串口日志

日志适合看：

```text
低频状态变化
fault 发生时的快照
参数命令
回零状态机
累计计数
```

不适合每 20 kHz 打印一次电流环变量。

高速日志会：

```text
扰乱实时性
堵塞串口
改变 bug 表现
```

如果要记录波形，应该做环形缓冲区，触发后批量导出，而不是快环里直接 printf。

## 8. 混叠问题

如果一个 20 kHz 信号用 5 Hz watch 看，看到的变化可能完全误导。

比如：

```text
真实 id/iq 高频抖动
watch 上只偶尔跳一下
或者看起来像慢速漂移
```

这就是抽样观察的混叠问题。

判断方法：

```text
watch 看趋势。
示波器看高频。
计数器看事件是否发生。
触发记录看故障前后短窗口。
```

## 9. 故障时应该保存什么

每次异常先记录：

```text
state
fault_reason
safe_state_count
encoder_reject_count
encoder_hold_count
speed_reject_count
iu/iv/iw/i_sum
id/iq
vd/vq
v_limited
duty_u/v/w
speed_fb_rpm
speed_iq_cmd
```

如果是回零问题，再记录：

```text
g_screw_axis_watch.home_state
g_screw_axis_watch.moving
g_screw_axis_watch.homed
g_screw_axis_watch.fault_seen
outer_encoder_pos
zero_encoder_pos
travel_counts
pos_counts
```

## 10. 观察手段选择表

| 问题 | 首选观察 |
| --- | --- |
| 状态机不进闭环 | Ozone watch |
| 一使能 fault | Ozone watch + fault 快照 |
| PWM 是否输出 | 示波器 + duty watch |
| 电流采样是否干净 | 示波器采样电阻 + i_sum watch |
| 电流环高频抖 | 示波器/DAC，watch 看趋势 |
| 速度低速抖 | watch + 低频日志 |
| 回零误判 | watch + 事件记录 |
| 母线掉压 | 示波器/万用表记录 VBUS |
