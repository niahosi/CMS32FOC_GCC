# 05 电流采样学习记录

本文记录当前 FOCAC6 调试中关于三相电流采样、相序映射、周期性波形和 JScope 判断方法的学习内容。[[99 学习与排错记录]] 只保留简短索引，详细解释放在本文，避免主学习记录继续膨胀。

## 1. 当前问题

当前已经确认：

```text
EPWM 可以输出 20 kHz
EPWM CMP0 可以触发 ADC 同步采样
MA600 可以提供电角度缓存
模式 4 IF 下电机可以转动
```

但在 JScope 中观察：

```text
g_elementary.current.iu
g_elementary.current.iv
g_elementary.current.iw
```

时，不同 raw 两相映射表现不同：

```text
VW：之前能看到明显周期包络，阶段性最像有效电流反馈
UV：三相看起来较均衡，但整体贴近零附近噪声，缺少明显周期结构
UW：V/W 有明显周期包络，U 仍然比较弱
```

这说明当前还不能只凭“电机能转”判断电流映射正确。模式 4 是 IF 开环角度加电流 PI，开环角度能拖动电机，但电流反馈映射仍可能不完全正确。

## 2. 为什么相电流应该有周期性

三相无刷电机有 U/V/W 三组绕组，它们在空间上相差约 120 度电角度。程序输出旋转电压或电流矢量时，电角度 `theta` 会不断变化。

理想情况下，相电流可以粗略理解为：

```text
iu = I * sin(theta)
iv = I * sin(theta - 120deg)
iw = I * sin(theta + 120deg)
```

所以只要转子在持续转动，并且采样映射正确，三相电流中应该能看到和电角度相关的重复变化。这个重复变化就是 JScope 图中的“周期性包络”。

实际波形不会是教科书那种干净正弦，原因包括：

```text
空心杯电机电感小，PWM 纹波更明显
低边采样受到开关噪声和采样窗口影响
模式 4 IF 的开环 theta 可能和真实转子位置有滑差
电流 PI 参数还未调稳
JScope 采样和快环变量刷新不是严格等价于示波器连续采样
```

因此当前不要要求 `iu/iv/iw` 一定是漂亮正弦。更重要的是看它们是否有和转动同步的周期成分、是否幅值合理、是否能让 `id/iq` 可控。

## 3. 为什么“看起来均衡”不一定正确

UV 图看起来较均衡，是因为三条线大多都在零附近抖动。这个现象有两种解释：

```text
解释 A：三相电流确实都很小而且均衡
解释 B：三相都没有采到明显有效电流，只是在看 offset 后的噪声
```

当前更偏向解释 B，是因为 UV 图缺少明显的周期结构。也就是说，它虽然“安静”，但没有证明这些变化来自电机旋转产生的有效相电流。

判断有效电流时，不只看三条线是否差不多大，还要看：

```text
1. 幅值是否随 if_iq_ref 或 vf_voltage 增大而增大
2. 是否有重复周期包络
3. id/iq 是否能跟随给定
4. vd/vq 是否合理，v_limited 是否很少触发
5. duty_u/v/w 是否没有长期贴边
```

## 4. 为什么某一相可能没有周期变化

如果某一相长期只在零附近抖，而另外两相有明显周期变化，不能直接说该相硬件坏。常见可能性有：

```text
1. raw 通道没有映射到正确的逻辑相
2. 当前两相采样组合没有选到最适合 FOC 的两路
3. 该相低边采样窗口不合适，采不到真实相电流
4. 该相 PGA/ADC/采样电阻/PCB 噪声链路和其他相不同
5. 当前电流幅值太小，有效成分被噪声盖住
6. 模式 4 开环 theta 和真实转子磁场有滑差，使电流分布不均
```

因此更准确的说法是：

```text
某相在当前映射和当前测试条件下不像有效主反馈。
```

不是：

```text
某相一定坏了。
```

## 5. 当前三组映射的含义

`Board_UpdateCurrent()` 中 raw 电流是 ADC/PGA 物理通道扣零漂后的值：

```text
raw U = s_iu_raw_cnt
raw V = s_iv_raw_cnt
raw W = s_iw_raw_cnt
```

logic 电流是 Motor/FOC 真正使用的相电流：

```text
logic Iu = Board_GetIuCnt()
logic Iv = Board_GetIvCnt()
logic Iw = Board_GetIwCnt()
```

当前采用两相采样、第三相重构。运行时 `EPWM CMP0` 只触发两相 ADC，未被触发的 raw 通道会被压回 offset，因此对应的 `*_raw_cnt` 会显示为 0；真正给 FOC 使用的是 logic 电流：

```text
Ia + Ib + Ic = 0
```

三组可选映射由 `Config.h` 中的 `CURRENT_SAMPLE_PAIR` 决定：

```text
UV:
logic Iu = raw U
logic Iv = raw V
logic Iw = -logic Iu - logic Iv

UW:
logic Iu = raw U
logic Iw = raw W
logic Iv = -logic Iu - logic Iw

VW:
logic Iv = raw V
logic Iw = raw W
logic Iu = -logic Iv - logic Iw
```

当前程序默认：

```text
CURRENT_SAMPLE_PAIR = CURRENT_SAMPLE_VW
```

注意：`CURRENT_SAMPLE_UVW_DIAG` 是测试模式，用于 CMP0 顺序采样 U/V/W 三相 raw 电流并直接作为 logic 电流观察。它不重构第三相，适合比较三相采样窗口和波形；正式 FOC 调试仍建议使用 `CURRENT_SAMPLE_UV/UW/VW` 中噪声最小、正负 iq 响应最对称的一组。

## 6. 如何用 JScope 判断

优先观察：

```text
g_elementary.current.iu
g_elementary.current.iv
g_elementary.current.iw
g_elementary.current.sum

g_elementary.current.iu_raw_cnt
g_elementary.current.iv_raw_cnt
g_elementary.current.iw_raw_cnt
g_elementary.current.raw_sum

g_elementary.foc.id
g_elementary.foc.iq
g_elementary.foc.vd
g_elementary.foc.vq
g_elementary.foc.v_limited

g_elementary.pwm.duty_u
g_elementary.pwm.duty_v
g_elementary.pwm.duty_w
```

比较三相电流时，先把 JScope 三个通道设置成相同显示比例，例如：

```text
Y Resolution = 20/div
Y Offset = 0
```

然后记录每相的峰峰值：

```text
pp = max - min
```

例如 UW 图中：

```text
iu: min -28, max 22  -> pp = 50
iv: min -31, max 67  -> pp = 98
iw: min -67, max 22  -> pp = 89
```

这个结果说明在 UW 映射下，V/W 变化幅度约为 U 的两倍，U 在当前条件下较弱。

## 7. 下一步验证方法

为了区分“幅值太小”和“映射不对”，可以保持同一个映射，只逐步增加 IF 电流给定：

```text
g_cmd.run.ctrl_mode = 4
g_cmd.open_loop.if_iq_ref = 10
g_cmd.open_loop.if_iq_ref = 20
g_cmd.open_loop.if_iq_ref = 30
g_cmd.open_loop.if_iq_ref = 40
```

每一档观察：

```text
iu/iv/iw 的 pp 是否一起增大
id/iq 是否可控
vq 是否明显增加
v_limited 是否频繁为 1
fault_reason 是否变化
```

判断：

```text
如果某相只是幅值小，给定增大后它应该出现更明显周期包络。
如果某相始终只在零附近抖，而另外两相明显增大，说明该相在当前映射下不适合作为有效主反馈。
```

最终选择映射时，优先级是：

```text
1. iq 能跟随 iq_ref
2. id 偏差不过大
3. v_limited 很少触发
4. 三相 logic 电流有周期结构且相对均衡
5. raw_sum 没有大异常
6. 电机能稳定同向转
```

“电机能转”只能说明开环拖动能工作，不能单独证明 FOC 电流映射正确。

## 8. 为什么 UV/VW/UW 三组 cnt 图形会不同

当前 JScope 里看到的 `g_elementary.current.iu/iv/iw` 不是固定等于 ADC 的 raw U/V/W，而是 `Board_UpdateCurrent()` 里映射后的 logic current。不同映射会把同一组 raw 电流重新组合成不同的 logic 三相。

三组映射分别是：

```text
UV:
logic Iu = raw U
logic Iv = raw V
logic Iw = -logic Iu - logic Iv

VW:
logic Iu = raw V
logic Iv = raw W
logic Iw = -logic Iu - logic Iv

UW:
logic Iu = raw U
logic Iv = raw W
logic Iw = -logic Iu - logic Iv
```

如果当前 raw 三相中只有某一路周期性最明显，例如 raw W 明显，而 raw U/raw V 较弱，那么映射后会出现：

```text
包含 raw W 的那一路 logic 电流会有明显周期。
由 -Iu-Iv 重构出来的第三相，会跟 raw W 大体反向。
没有包含明显 raw 周期的那一路，会更像零附近噪声。
```

这就是为什么有些图看起来是一正一反：不是程序凭空制造了反向电流，而是两相重构公式要求三相和接近 0。

```text
Iu + Iv + Iw = 0
如果 Iu 很小，则 Iw ≈ -Iv
```

因此一张图中长期出现“两相一正一反、另一相很小”，说明当前 logic 三相更像“两相有效 + 一相弱”，还不是理想三相轮换。

判断时要分两步：

```text
1. 先看 raw U/V/W：确认当前被 CMP0 触发的两路真实 ADC/PGA 通道里谁有周期。未采样那一路 raw_cnt 为 0 是正常现象。
2. 再看 logic Iu/Iv/Iw：确认映射后 FOC 看到的三相是否均衡。
```

如果 raw 本身只有一相明显，继续切 UV/VW/UW 只能改变“哪条 logic 曲线显得明显”，不能从根本上得到理想三相电流。此时应优先排查采样窗口、ADC/PGA 通道、PWM duty 与触发点关系，而不是只改映射。

## 9. 为什么三相扫描时只有一相明显，CMP0 两相采样后能看到两相正弦

这个现象的核心不是“三电阻硬件不如两电阻”，而是运行时采样是否真的发生在同一个 PWM 有效窗口里。

本板硬件仍然是三电阻、三路 PGA：

```text
U: PGA0 -> ADC_CH_0
V: PGA1 -> ADC_CH_2
W: PGA2 -> ADC_CH_3
```

但 FOC 运行时不一定适合在一个触发点里把三相全部扫完。低边电流采样有一个很现实的限制：只有某一相下桥臂导通、采样电阻上确实流过相电流，并且开关噪声已经稳定后，这一相的 ADC 值才有意义。

以前“三相都采”的问题可能是：

```text
EPWM CMP0 触发一次
ADC 按通道顺序扫描 U/V/W
第一相在比较合适的窗口里
第二、第三相已经偏离窗口，或者落到开关噪声/无有效低边电流区域
```

这样 JScope 里就会看到：

```text
只有一相有明显周期变化
另外两相像噪声、小幅抖动或不跟随转动
```

改成“CMP0 对应两相电流”后，程序做了两件关键的事：

```text
1. CURRENT_ADC_CH_MASK 只打开选中的两路 ADC 通道
2. ADC 中断只等这两路里的最后一路完成
```

例如当前 `CURRENT_SAMPLE_PAIR = CURRENT_SAMPLE_VW` 时：

```text
CMP0 只触发 ADC_CH_2 和 ADC_CH_3
raw V = ADC_CH_2
raw W = ADC_CH_3
raw U = offset
logic U = -logic V - logic W
```

这样做的好处是：

```text
转换序列更短，两路更接近同一个 PWM 采样时刻
两路更容易落在同一个稳定采样窗口内
未采样那一相不再拿一个“可能无效的 ADC 值”参与 FOC
第三相由 Ia + Ib + Ic = 0 重构，数学上更干净
```

所以你现在能找到两相正弦，是因为我们把运行采样从“试图一次扫三相”收敛为“在同一个 CMP0 窗口内只采最可靠的两相”。这和很多电机控制工程的思路一致：三电阻硬件可以都存在，但实时 FOC 可以只取当前最可信的两相，第三相重构。

需要注意：

```text
raw U/V/W 是物理 ADC/PGA 通道
logic Iu/Iv/Iw 是 FOC 使用的三相电流
未被 CMP0 触发的 raw 通道显示为 0 附近是正常现象
判断闭环要看 logic Iu/Iv/Iw、id/iq、vd/vq、v_limited
```

一句话总结：

```text
以前三相扫描像是在一个很短的安全窗口里塞了太多转换；
现在两相同步采样减少了转换时间误差，避开无效窗口，再用三相和为 0 重构第三相。
```

## 10. 三相 CMP0 诊断模式

为了测试三电阻硬件本身和三相 ADC/PGA 链路，当前加入了一个明确的诊断配置：

```text
CURRENT_SAMPLE_UVW_DIAG
```

这个模式下：

```text
CMP0 触发 ADC_CH_0 / ADC_CH_2 / ADC_CH_3
raw U = ADC_CH_0 - U offset
raw V = ADC_CH_2 - V offset
raw W = ADC_CH_3 - W offset
logic Iu = raw U
logic Iv = raw V
logic Iw = raw W
raw_sum = raw U + raw V + raw W
logic sum = logic Iu + logic Iv + logic Iw
```

它和正式两相重构模式的区别：

```text
两相模式：采样更短、更同步，第三相由 Ia+Ib+Ic=0 重构，适合闭环。
三相诊断：三路 raw 都能看到，但扫描时间更长，可能重新暴露采样窗口偏移，适合测试。
```

使用它时重点观察：

```text
g_elementary.current.iu_raw_cnt
g_elementary.current.iv_raw_cnt
g_elementary.current.iw_raw_cnt
g_elementary.current.raw_sum

g_elementary.current.iu
g_elementary.current.iv
g_elementary.current.iw
g_elementary.current.sum
```

如果三相 raw 都有可解释的周期变化，说明硬件链路基本通；如果只有一相或两相明显，说明第三相可能仍然落在不合适的采样窗口，不能直接拿来作为正式 FOC 闭环输入。

## 11. VW 两相采样波形不够干净时怎么调

当前 `CURRENT_SAMPLE_PAIR = CURRENT_SAMPLE_VW` 时：

```text
raw U = offset，因此 iu_raw_cnt 长期为 0 是正常现象。
raw V = ADC_CH_2，是真实采样相。
raw W = ADC_CH_3，是真实采样相。
logic U = -logic V - logic W，是重构相。
```

因此在 JScope 中看两相采样质量时，不要用绿色 `iu_raw_cnt` 判断 U 相是否正常；重点看：

```text
g_elementary.current.iv_raw_cnt
g_elementary.current.iw_raw_cnt
g_elementary.current.iu
g_elementary.current.iv
g_elementary.current.iw
g_elementary.foc.id / iq
g_elementary.foc.vd / vq / v_limited
```

红蓝两相有周期变化，说明采样已经能跟随电机电流；但尖峰、凹口、局部变形说明采样点、PWM 扇区、开关噪声、低边导通窗口和开环拖动状态仍在影响波形。

优先调整顺序：

```text
1. 扫 PWM_ADC_TRIGGER_TICK_DEFAULT：650 / 750 / 850 / 900 / 950。
2. 每次只改一个采样点，保持同样的 VF/IF 给定和 JScope 比例。
3. 比较 iv_raw_cnt / iw_raw_cnt 的尖峰数量、峰峰值、凹口深度和周期连续性。
4. 同时看 logic Iu/Iv/Iw 是否三相更平衡。
5. 正式闭环前再看 id/iq 是否跟随、v_limited 是否少触发。
```

不要急着用强滤波把图“抹平”。显示用 Moving Average 可以开一点帮助观察趋势，但控制用电流如果靠强滤波才稳定，通常说明采样点、相序、零漂或电角度还有问题。

## 12. 电流采样中可以用哪些滤波

电流采样滤波要先区分用途：

```text
显示滤波：让 JScope/FreeMASTER 图更容易看，不参与控制。
控制滤波：进入 Clarke/Park/PI 的电流反馈，必须非常轻，不能引入明显延迟。
保护滤波：用于过流判断，可以短时间去尖峰，但不能把真实过流滤掉。
```

推荐优先级：

```text
1. 先调采样点和采样窗口，不先靠滤波解决。
2. 显示变量可以用 Moving Average 或一阶 IIR。
3. 控制变量最多使用很轻的一阶 IIR 或尖峰限幅。
4. 过流保护用连续计数或轻微去尖峰，不用重低通。
```

可选滤波方式：

```text
一阶 IIR：
  y += (x - y) >> n
  优点：计算轻，适合 CMS32 定点。
  缺点：n 越大延迟越大。控制电流建议 n=1 或 2 起步，显示可用 n=3~5。

限幅去尖峰：
  如果本次采样相对上次跳变超过阈值，就限制到最大允许跳变量。
  优点：对偶发尖峰有效，延迟小。
  缺点：阈值太小会削掉真实电流变化。

中值滤波：
  取最近 3 点中间值。
  优点：对孤立尖峰有效。
  缺点：实现和状态多一点，且有延迟；当前不优先放进快环。

滑动平均：
  N 点平均。
  优点：显示好看。
  缺点：延迟大；不建议直接用于电流环 PI 输入。

硬件 RC / ADC 采样时间：
  从模拟端减少高频尖峰。
  优点：在 ADC 前处理噪声。
  缺点：硬件 RC 太大也会拖慢真实电流变化；需要结合 PWM 频率和采样电阻/PGA 带宽。
```

当前 FOCAC6 更建议：

```text
JScope 显示：先用 JScope 自带 Moving Average 看趋势。
控制输入：先不滤，或只加很轻的一阶 IIR。
尖峰问题：优先扫 PWM_ADC_TRIGGER_TICK_DEFAULT，再考虑限幅去尖峰。
```

需要记住：电流环希望“快而真实”，不是“图像最平滑”。如果滤波让 `id/iq` 看起来平滑但 `vq/vd` 反应变慢，电机会更容易抖。

## 13. 为什么相名不对应但 dq 轴可能是对的

2026-06-15 追加这个小节，是因为它非常容易误解。

当前调试中可能看到：

```text
ADC 物理通道 raw U/V/W
  和
FOC 使用的 logic Iu/Iv/Iw
```

名字不完全对应，但 `id/iq` 反而落在比较正确的轴上。这个现象并不矛盾。

FOC 不直接关心丝印相名，它关心三套坐标是否自洽：

```text
1. 电流采样映射
2. SVPWM 输出相序
3. 电角度零点和方向
```

例如一种历史/当前常见映射是：

```text
logic Iu = raw V
logic Iv = raw W
logic Iw = -raw V - raw W
```

如果真实三相满足：

```text
raw U + raw V + raw W = 0
```

那么：

```text
logic Iw = raw U
```

所以 logic 三相等价于：

```text
logic U = physical V
logic V = physical W
logic W = physical U
```

这相当于三相名字整体循环挪了一位，在 Clarke/Park 里等价于 alpha-beta 坐标整体旋转一个固定角度。只要电角度零点或输出相序也在同一套旋转坐标里，`id/iq` 仍然可能是对的。

因此判断时不要只问：

```text
logic U 是不是物理 U？
```

还要问：

```text
logic 电流坐标、SVPWM 输出坐标、电角度零点是否一起自洽？
```

真正危险的是只修正其中一个环节。例如只把 logic U/V/W 改回物理 U/V/W，但不重新标定电角度零点和 q 轴方向，可能会把原来能工作的坐标系拆散，导致 `id/iq` 串轴、正负 q 轴不对称或电流环半正反馈。

当前建议：

```text
raw U/V/W 用来查硬件采样。
logic Iu/Iv/Iw 用来查 FOC 当前控制坐标。
id/iq 用来判断当前控制坐标是否与电角度自洽。
```

后续若要统一命名，应在同一次验证里同时对齐：

```text
EPWM U/V/W
ADC raw U/V/W
电机物理线 U/V/W
logic Iu/Iv/Iw
电角度零点和方向
```

## 14. 清理测试程序后波形变差的排查记录

2026-06-10 对比发现，“删除测试钉子”本身不会让 ADC 波形变差，但这次清理前后确实有几处会改变波形的实质性差异：

```text
1. 采样点变了：
   当前 PWM_ADC_TRIGGER_TICK_DEFAULT = 660。
   早期常用过 800/900/950 等点，用户曾观察 900/950 噪声更小。

2. 观察变量变了：
   当前截图看的是 iv_raw_cnt / iw_raw_cnt，raw 是物理 ADC/PGA 通道。
   早期很多图看的是 logic Iu/Iv/Iw，logic 可能包含重构相，视觉上更像三相正弦。

3. 逻辑相位命名变了：
   早期固定映射等价于 logic U = raw V, logic V = raw W, logic W = -rawV-rawW。
   当前 CURRENT_SAMPLE_VW 是 logic V = raw V, logic W = raw W, logic U = -rawV-rawW。
   两者使用同样 raw V/W，但显示到 U/V/W 名字上的相位不同。

4. PWM/电流限制默认值变了：
   PWM_DUTY_MIN/MAX 从较窄的 700/900 放宽到 300/1300。
   MOTOR_CURRENT_V_LIMIT 和 MOTOR_CURRENT_REF_LIMIT 从 80 放宽到 160。
   如果实际输出幅值或占空比范围变大，采样窗口和电流纹波都会变化。

5. 开环模块拆分了：
   VF/IF 从 Motor_Loop.c 移到 Motor_OpenLoop.c，理论公式基本一致，但默认速度、超时、快照来源和 PI 初始化路径发生了整理。
```

优先回退验证顺序：

```text
1. 先把 PWM_ADC_TRIGGER_TICK_DEFAULT 调回旧图对应的采样点，例如 900 或 950。
2. 同时观察 raw V/W 和 logic Iu/Iv/Iw，不要用 raw 图直接和以前 logic 图比较。
3. 如果要复现早期相位显示，可临时用旧固定映射：logic U=rawV, logic V=rawW, logic W=-rawV-rawW。
4. 确认 g_cmd.open_loop 的速度、电压、电流给定和之前完全一致。
5. 若波形仍差，再比较 PWM_DUTY_MIN/MAX 与 MOTOR_CURRENT_V_LIMIT 是否影响实际输出。
```

## 15. VW 两相 raw 电流曲线怎么看

2026-06-11 用户观察 `iv_raw_cnt` 和 `iw_raw_cnt` 两条曲线，现象为：

```text
两条曲线都围绕 0 上下变化
幅值接近，约正负 70~90 count
两条曲线不是反相，而是错开约 120 电角度
曲线有 PWM/ADC 噪声毛刺，但整体周期连续
```

在 `CURRENT_SAMPLE_PAIR = CURRENT_SAMPLE_VW` 下，这是比较符合预期的形态。当前程序实际采样：

```text
Iv = iv_raw_cnt
Iw = iw_raw_cnt
Iu = -Iv - Iw
```

所以只看两条 raw 曲线时，正确趋势不是“一条等于另一条的负数”，而是两相三相正弦中的两相，理论相位差约 120 度。第三相由软件重构出来。

判断步骤：

```text
1. 看两条 raw 是否都以 0 为中心。
2. 看两条 raw 幅值是否接近。
3. 看两条 raw 是否大约相差 120 度，而不是同相或完全反相。
4. 同时打开 logic current：current.iu / current.iv / current.iw。
5. logic 三相应呈三条相位错开的波形，sum 应接近 0。
```

按当前电流换算：

```text
1 count ≈ 5.49 mA
80 count ≈ 0.44 A
160 count p-p ≈ 0.88 A p-p
```

注意：这张图只能说明“VW 两相采样形态基本合理”，不能单独证明：

```text
FOC 电角度零点正确
q 轴方向正确
电流符号完全正确
电流环 PI 参数合适
```

这些还要继续看 `id/iq/vd/vq/duty_u/v/w` 和电机正反响应。
