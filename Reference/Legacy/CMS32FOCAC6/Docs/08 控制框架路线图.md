# 08 控制框架路线图

本文记录 `CMS32FOCAC6` 从当前阶段继续实现 FOC、三闭环、定点运算和弱磁的方案。目标是既保留清晰分层，又不把工程拆得过碎。

当前硬件对象是：

```text
CMS32M6513AGE40NB 控制板
空心杯无刷电机
滚珠丝杠负载
MA600 + 侧边空心磁环
三电阻低边电流采样
20 kHz EPWM
EPWM CMP0 触发 ADC
```

## 1. 当前代码基线

当前已经具备：

- `Board_Init()`：时钟、GPIO、MA600 SPI、ADC/PGA、EPWM、电流零漂、PWM 触发 ADC。
- `ADC_IRQHandler()`：调用 `Board_HandleAdcIrq()`，成功后进入 `Motor_FastLoop()`。
- `Board_Analog`：三相电流 raw、offset 后 count、三相和、ADC 同步计数。
- `Board_PWM`：20 kHz PWM、死区、三相 duty、输出使能/关断。
- `Board_MA600`：提供阻塞式读角和缓存读角两条路径，正式控制优先使用缓存。
- `Motor`：状态机、安全检查、同步角度缓存、电流环、速度估算和速度环。
- `App_Debug`：观察变量，角度、FOC、电流环和速度环调试读取 Motor 缓存，不直接读 MA600 SPI。

当前还没有：

- 最终电角度零点标定流程。
- 最终速度估算滤波参数。
- 最终电流环 PI 参数。
- 最终 SVPWM 电压矢量限幅。
- 速度环参数整定。
- 丝杠位置环。
- 弱磁控制。

当前速度环先行实现已经落地，详细说明见：

```text
[[04 三闭环控制设计]]
```

## 2. 目录和文件规划

短期先不拆太多文件，避免工程变散。

当前阶段继续保持：

```text
Motor/Inc/Motor.h
User/Motor/Src/Motor.c
```

当 `Motor.c` 明显变长、快环数学和状态机混在一起后，再按需要拆成：

```text
Motor/Inc/Motor.h
Motor/Inc/Motor_Foc.h
Motor/Inc/Motor_Profile.h
User/Motor/Src/Motor.c
Motor/Src/Motor_Foc.c
Motor/Src/Motor_Profile.c
```

拆分标准：

- `Motor.c`：状态机、模式切换、慢任务、保护、对外接口。
- `Motor_Foc.c`：快环数学，Clarke/Park、PI、SVPWM、角度换算。
- `Motor_Profile.c`：丝杠位置轨迹、限位、回零、速度规划。

不要新增顶层 `Control`、`Encoder`、`Bsp` 目录。MA600 仍属于 `Board`，FOC 使用的角度处理属于 `Motor`。

## 3. 核心数据结构建议

后续不要继续增加散乱全局变量，优先用结构体收口。

当前执行规范：

- `Board_Analog` 保存 ADC 原始值、零漂和 offset 后的电流 count，它是板级采样源。
- `Motor` 不再复制一组同名的 `s_iu_cnt/s_iv_cnt/s_iw_cnt`。
- `Motor` 内部用用途命名区分数据，例如 `s_check_cur` 表示安全检查快照，`s_loop` 表示闭环快环快照。
- 快环中会跨周期观察、调试、保护或参与下一步控制的量，放入 `s_loop` 结构体；只在几行内使用的纯中间量才使用局部变量。
- 闭环快环已经接入 SVPWM 输出，只有 `debug_motor_enable = 1` 且 `debug_ctrl_mode` 为 `MOTOR_CTRL_CURRENT` 或 `MOTOR_CTRL_SPEED` 时才允许输出；`MOTOR_CTRL_OFF` 必须关 PWM。

结构体推荐形态：

```c
typedef struct
{
    int16_t iu;
    int16_t iv;
    int16_t iw;
    int16_t sum;
} MotorCurrentCnt_t;

typedef struct
{
    uint16_t raw;
    uint16_t filt;
    uint16_t elec;
    int16_t delta;
    int32_t unwrap;
} MotorAngle_t;

typedef struct
{
    int16_t id;
    int16_t iq;
    int16_t vd;
    int16_t vq;
    int16_t duty_u;
    int16_t duty_v;
    int16_t duty_w;
} MotorFoc_t;

typedef struct
{
    int32_t ref;
    int32_t fb;
    int32_t err;
    int32_t integ;
    int32_t out;
    int32_t kp;
    int32_t ki;
    int32_t out_min;
    int32_t out_max;
} MotorPi_t;
```

说明：

- `CurrentCnt` 先保留 ADC count 单位，避免过早引入浮点。
- `AngleRaw` 表示 MA600 原始读数。
- `AngleFilt` 表示滤波后的角度。
- `AngleElec` 表示给 Park 变换使用的电角度。
- `unwrap` 用于速度和丝杠位置估算。
- PI 内部使用 `int32_t`，给积分留余量。

当前代码已经采用的最小闭环快环对象是：

```text
MotorLoop_t
  active
  output_ready
  angle_raw / angle_elec
  cur.iu / cur.iv / cur.iw / cur.sum
  i_alpha / i_beta
  id / iq
  id_ref / iq_ref
  vd / vq
  v_alpha / v_beta
  fast_count
```

这个设计的目的不是为了“多包一层”，而是为了避免电流环继续出现一堆局部变量和重复全局变量：

```text
Board_Analog 的 iu/iv/iw = 板级最新采样源
Motor s_loop.cur        = 本次快环使用的采样快照
App g_debug             = 对外观察快照
```

三者不能混名，也不能互相直接写。

## 3.1 状态表和回调的引入时机

当前 `Motor` 的状态数量还少，`switch (s_state)` 状态机比函数指针表更直观。

后续状态变多后，可以改成：

```text
MotorState_t 状态枚举
MotorStateNode_t 状态表
Enter / Run / Exit 三类状态函数
```

推荐拆分示意：

```text
MOTOR_STATE_IDLE
  EnterIdle()
  RunIdle()
  ExitIdle()

MOTOR_STATE_SENSOR_CHECK
  EnterSensorCheck()
  RunSensorCheck()
  ExitSensorCheck()

MOTOR_STATE_CLOSED_LOOP
  EnterClosedLoop()
  RunClosedLoop()
  ExitClosedLoop()

MOTOR_STATE_FAULT
  EnterFaultState()
  RunFaultState()
  ExitFaultState()
```

适合引入函数指针表的条件：

- 状态超过约 `5` 个。
- 每个状态都有明确的进入动作、周期动作、退出动作。
- `Motor_TASK()` 开始超过约 `120` 行。
- 新增状态时，经常要在多个 `case` 之间上下跳。

不适合引入函数指针表的情况：

- 只有两三个状态。
- 每个状态只有几行代码。
- 函数指针只是把直接调用变成间接调用，没有减少复杂度。

在快环中也可以使用少量函数指针，但只适合“算法策略选择”，例如：

```text
普通 SVPWM
零序注入 SVPWM
弱磁控制
测试用零电压输出
```

不建议把每一个底层硬件动作都做成回调。`Board_PWM`、`Board_Analog`、`Board_MA600` 当前都是本 PCB 的唯一硬件实现，直接调用更清楚。

## 4. 快环和慢环节拍

当前 PWM 是：

```text
SystemCoreClock = 64 MHz
PWM_FREQ_HZ = 20 kHz
PWM_PERIOD = 1600
ADC 由 EPWM CMP0 触发
```

建议阶段频率：

| 阶段 | PWM | 电流环 | 速度环 | 位置环 | 说明 |
| ---- | --: | -----: | -----: | -----: | ---- |
| 起步 | 20 kHz | 10 kHz | 500 Hz | 100 Hz | 快环先隔次执行，验证耗时和稳定性 |
| 目标 | 20 kHz | 20 kHz | 1 kHz | 200 Hz | 每次 ADC 同步采样都跑电流环 |
| 扩展 | 25 kHz | 25 kHz | 1 kHz | 200 Hz | 只有在定点优化和示波器验证后再尝试 |

推荐调度：

```text
ADC_IRQHandler
  -> Board_HandleAdcIrq()
  -> Motor_FastLoop()
       每次进入：采样快照、保护、角度更新
       每 1 次：电流环，目标 20 kHz
       每 20 次：速度环，约 1 kHz
       每 100 次：位置环，约 200 Hz

main while
  -> Motor_TASK()
       状态机、调试命令、故障复位、非实时任务
  -> Debug_UpdateSnap()
```

注意：

- 快环内禁止 `printf`、RTT 格式化、延时、阻塞 SPI 等耗时操作。
- MA600 SPI 是否每个快环都读，要先测耗时。如果读角太慢，先采用角度缓存或降低读取频率。
- 速度环和位置环不要放在主循环延时里，最好由快环分频产生稳定节拍。

### 4.1 2026-06-01 快环负载问题后的调度修正

实测现象：

- `debug_motor_enable = 1` 后，PWM/电流路径有动作，但主循环中的 `Motor_TASK()` 看起来不再进入。
- `debug_motor_enable = 0` 的关断链路补强后，PWM 可以关闭。
- 将快环降频后，主循环可以恢复。

结论：

```text
问题重点不是状态机不运行
而是 ADC 中断 + Motor_FastLoop 负载过高
导致 main while 被中断长时间挤占
```

因此后续调度要按“实时优先级”重新收口：

```text
ADC_IRQHandler
  只做必要采样和极短保护
  不做调试输出
  不做阻塞 SPI
  不做每次都需要多次除法的复杂计算

Motor_FastLoop
  先分频执行
  逐步恢复 Clarke/Park、PI、SVPWM
  每恢复一部分都观察 main_count 是否仍增长

Motor_TASK
  状态机、使能/禁用、故障复位、进入闭环条件
  可以读较慢的传感器缓存
  不承担 20 kHz 精确采样

Debug_UpdateSnap
  只读取缓存变量
  不主动发起 MA600 SPI 事务
```

建议新的阶段频率：

| 阶段 | ADC/PWM | 电流环计算 | MA600 读取 | 主循环 | 目标 |
| ---- | ------: | ---------: | ---------: | -----: | ---- |
| 定位 | 20 kHz | 0 ~ 2 kHz | 主循环/缓存 | 必须稳定 | 确认主循环不再被饿死 |
| 恢复 | 20 kHz | 5 kHz | 1 ~ 5 kHz | 必须稳定 | 逐步恢复 FOC 模块 |
| 优化 | 20 kHz | 10 kHz | 实测决定 | 必须稳定 | 验证耗时和电流稳定性 |
| 目标 | 20 kHz | 20 kHz | 需实测确认 | 必须稳定 | 只有在快环预算足够后进入 |

设计规则：

- 快环每增加一个功能，都要观察 `main_count`、`adc_irq_count`、`fastloop_count`。
- 如果 `main_count` 停止或明显变慢，说明中断负载已经过高。
- MA600 的 SPI 读角应先缓存化，调试显示读取缓存，不重复读硬件。
- `Board_EnablePwmOutput()` 不应在每次快环里调用，输出使能应由状态机管理。

当前已落地的改动：

- `Board_UpdateAngle()` 负责触发一次 MA600 SPI 读角，并更新 `Board_MA600` 内部缓存。
- `Board_GetAngleRaw()` / `Board_IsAngleOk()` 只读缓存，不发起 SPI 事务。
- `Motor_TASK()` 通过 `UpdateAngle()` 更新 raw、elec、delta、pos、ok、age。
- `Motor_FastLoop()` 只读取 `s_angle.elec` 缓存，不再直接读 MA600 SPI。
- `angle_age` 在快环中递增，主循环成功更新 MA600 后清零；如果角度缓存过旧，快环进入故障保护。

## 5. MA600 角度处理

本项目的关键点：

- 电机极对数是 4。
- 磁环极对数按 4 处理。
- 已观察到机械转一圈 MA600 角度周期出现 4 次。

因此：

```text
FOC 电角度可以直接使用校正后的 MA600 磁场角
机械角/丝杠位置需要把解包后的磁场角圈数除以磁环极对数
```

推荐流程：

```text
Motor_TASK
  -> Board_UpdateAngle
  -> MA600 raw cache
  -> 方向修正
  -> 零点修正
  -> 可选滤波
  -> angle_elec 缓存给 FOC
  -> unwrap 后除以 magnet_pole_pairs 得到机械圈数
  -> 机械圈数换算丝杠位置

Motor_FastLoop
  -> 读取 angle_elec 缓存
  -> 不直接发起 MA600 SPI
```

公式：

```text
angle_dir = raw 或 -raw
angle_elec = angle_dir - electrical_zero_offset

sensor_elec_turns = unwrap(angle_elec) / 65536
motor_mech_turns = sensor_elec_turns / magnet_pole_pairs
linear_pos_mm = motor_mech_turns * screw_lead_mm / gear_ratio
```

如果未来更换成单对极磁编码器，才使用：

```text
angle_elec = mechanical_angle * motor_pole_pairs + zero_offset
```

不要把这两个场景混在一起。

当前角度缓存字段：

| 字段 | 含义 |
| ---- | ---- |
| `raw` | MA600 最近一次成功读取的 16 位原始角 |
| `elec` | 方向和零点修正后的 FOC 电角度 |
| `delta` | 相邻两次 raw 的 16 位差值，用于解缠 |
| `pos` | 按 MA600 周期累计的位置；换算机械位移时还要除以磁环极对数 |
| `ok` | 最近一次 MA600 更新是否成功 |
| `age` | 角度缓存年龄；快环每使用一次递增，主循环更新成功后清零 |

## 6. 三闭环设计

三闭环从内到外是：

```text
电流环 -> 速度环 -> 位置环
```

### 电流环

输入：

```text
iu/iv/iw
angle_elec
id_ref
iq_ref
```

输出：

```text
duty_u/duty_v/duty_w
```

流程：

```text
iu/iv/iw
  -> Clarke
  -> Park
  -> id/iq
  -> PI(id)、PI(iq)
  -> vd/vq
  -> 电压限幅和抗积分饱和
  -> inverse Park
  -> SVPWM 或零序注入
  -> Board_SetPwmDuty()
```

起步建议：

- `id_ref = 0`。
- `iq_ref` 先由调试变量给一个很小值。
- 先计算不输出，确认 `id/iq` 合理后再输出。
- 电机带丝杠，所有输出都要短时间、低电流、可立即关断。

### 速度环

输入：

```text
speed_ref
speed_fb
```

输出：

```text
iq_ref
```

速度来源：

```text
MA600 angle unwrap
  -> 差分
  -> IIR 低通
  -> rpm 或 q 格式速度
```

建议：

- 速度环先 500 Hz，再到 1 kHz。
- `iq_ref` 必须限幅。
- 加积分限幅和输出限幅。
- 位置模式下速度目标由轨迹规划器生成，不要直接阶跃给速度环。

### 位置环

本电机带滚珠丝杠，位置环要比普通旋转电机更重视安全。

输入：

```text
pos_ref_mm
pos_fb_mm
```

输出：

```text
speed_ref
```

必须设计：

- 上电回零或用户确认零点。
- 软件正负限位。
- 最大速度、最大加速度。
- 位置跟随误差保护。
- 堵转/卡住保护。
- 运动命令必须是有限位移，不允许无限单方向开环跑。

轨迹建议：

```text
位置目标
  -> 梯形速度规划或简单 S 曲线
  -> speed_ref
  -> speed PI
  -> iq_ref
```

第一版可先做梯形速度规划，等机械表现稳定后再做 S 曲线。

## 7. 定点运算方案

CMS32M6513 是 Cortex-M0+，没有 FPU。后续快环建议使用定点。

推荐格式：

| 变量 | 格式 | 说明 |
| ---- | ---- | ---- |
| 角度 | `uint16_t` | 0 到 65535 表示 0 到 2π |
| sin/cos | Q15 | 查表输出 `-32768..32767` |
| 电流 count | `int16_t` | ADC offset 后的 count |
| 电流 A | Q12 或 Q15 | 只在需要物理量时换算 |
| id/iq | Q15 或 count | 起步可直接用 count 做 PI |
| vd/vq | Q15 | 归一化电压，`32767` 约等于最大可用电压 |
| PI 积分 | `int32_t` | 防止积分溢出 |
| duty | `uint16_t` | 直接对应 PWM compare count |

电流换算参考：

```text
I[A] = adc_cnt * Vref / 4096 / Rshunt / PGA
```

按当前代码配置估算：

```text
Vref = 3.6 V
Rshunt = 0.08 ohm
PGA = 2

1 count ≈ 3.6 / 4096 / 0.08 / 2
        ≈ 0.00549 A
```

快环里不要每次做浮点除法。建议提前算成定点比例：

```text
current_mA = adc_cnt * 549 / 100
```

或使用 Q 格式比例系数。

第一版可以这样取舍：

- 快环：定点或 count 单位。
- 慢环/调试显示：允许少量浮点换算，方便理解。
- 文档和参数表保留真实物理单位，代码内部再转成定点。

## 8. PI 参数和电机参数

电流环 PI 依赖电机电阻、电感和采样频率。

常见连续域估算：

```text
Kp = L * wc
Ki = R * wc
```

但当前参数表里的电感写成：

```text
ls_phase = 19 H
```

这个单位非常可疑。小型空心杯无刷电机通常不应直接按 `19 H` 使用。后续必须确认它到底是：

```text
19 H
19 mH
19 uH
```

在单位确认前，不要按这个电感计算最终 PI。可以先用很小的经验参数，在低电流、短时间下调试。

## 9. 弱磁设计

弱磁不是下一步必须做的功能，建议默认关闭。

启用弱磁的前提：

- 电流环稳定。
- 速度环稳定。
- 电角度零点正确。
- 电流方向正确。
- 已知道母线电压和最大安全电流。
- 速度接近基础转速，电压利用率已经接近上限。

弱磁基本思路：

```text
正常区：
  id_ref = 0
  iq_ref 由速度环给出

电压饱和且需要更高速度：
  id_ref 逐步变为负值
  限制 sqrt(id_ref^2 + iq_ref^2) 不超过电流上限
  降低 iq_ref 上限，避免总电流超限
```

对本项目要特别保守：

- 电机带丝杠，大多数场景追求可控位置和推力，不是高速空转。
- 弱磁会增加电流和发热，并降低同电流下的转矩能力。
- 位置模式中不建议为了追位置误差激进弱磁。

建议实现顺序：

```text
先保留 weak_field_enable = 0
先保留 id_fw_min、voltage_limit 等参数
等三闭环稳定后再实现弱磁调节器
```

## 10. 状态机规划

当前状态机已清理为：

```text
IDLE
SENSOR_CHECK
CLOSED_LOOP
FAULT
```

后续建议在 `CLOSED_LOOP` 内按模式演进为：

```text
IDLE
SENSOR_CHECK
ALIGN
CURRENT_LOOP
SPEED_LOOP
POSITION_LOOP
FAULT
```

说明：

- `SENSOR_CHECK`：确认 MA600、零漂、电流和 PWM 安全状态。
- `ALIGN`：短时间小电流定向，标定或验证电角度零点。
- `CURRENT_LOOP`：只闭电流环，用调试量给 `iq_ref`。
- `SPEED_LOOP`：速度环输出 `iq_ref`。
- `POSITION_LOOP`：丝杠位置环输出 `speed_ref`。
- `FAULT`：关闭输出，等待明确复位。

早期 `OPEN_LOOP` 虚拟正弦输出路径已经删除。后续如果需要对齐或小信号输出，应作为明确的 `ALIGN` 或 `CURRENT_LOOP` 子模式实现，并且带限时、限流、限位。

## 11. 安全保护清单

必须优先做：

- 输出默认关闭。
- 进入任何运行态前确认 MA600 有更新。
- 进入任何运行态前确认三相电流在零漂附近。
- `iu + iv + iw` 超限保护。
- 单相电流 count 超限保护。
- PWM 输出超时自动关断。
- FAULT 状态必须关断 PWM。
- 调试命令不能绕过 FAULT。

三闭环前必须补：

- 电流环输出限幅。
- PI 积分限幅。
- duty 限幅。
- 角度跳变保护。
- MA600 通信失败保护。
- 丝杠软件限位。
- 位置跟随误差保护。
- 堵转或过载保护。

后续可补：

- NTC 过温保护。
- VM 欠压/过压保护。
- 硬件 ACMP 过流保护复核。
- 看门狗。

## 12. 实现里程碑

### M1：快环骨架

目标：

- `Motor_FastLoop()` 中读取电流快照。
- 增加快环计数和分频计数。
- 在快环中只做保护判断，不输出新 duty。

验收：

- `adc_sync_count` 增加。
- 快环计数增加。
- 主循环仍可正常刷新 `g_debug`。
- PWM 输出关断时电流保持零漂附近。

### M2：角度和速度估算

目标：

- 区分 `AngleRaw`、`AngleFilt`、`AngleElec`。
- 实现角度 unwrap。
- 实现速度估算。
- 根据 `magnet_pole_pairs = 4` 得到机械圈数和丝杠位置。

验收：

- 手动转动丝杠或转子时，角度方向、速度方向、位置方向一致。
- 静止速度接近 0。
- 角度跳变不会让速度爆掉。

### M3：FOC 数学只计算不输出

目标：

- Clarke。
- Park。
- sin/cos 查表。
- `id/iq` 计算。
- PI 结构体和限幅。
- SVPWM duty 计算。

验收：

- 不打开 PWM 输出也能观察计算结果。
- 静止无电流时 `id/iq` 在 0 附近。
- duty 计算结果在安全范围内。

### M4：电流环小信号输出

目标：

- 用很小的 `iq_ref` 做短时间电流环输出。
- 所有输出受闭环状态机、输出允许标志、限时和故障保护控制。

验收：

- 输出可立即关闭。
- 电流方向可解释。
- 不出现明显过流。
- 丝杠不被长时间单方向驱动。

### M5：速度环

目标：

- 速度环 500 Hz 到 1 kHz。
- 速度目标由调试变量给出，但必须限幅。
- 输出 `iq_ref`。

验收：

- 正负小速度能短距离往返。
- 停止命令能回到零电流附近。
- 速度环积分不会越积越大。

### M6：位置环和丝杠轨迹

目标：

- 建立机械零点。
- 加软件限位。
- 实现梯形速度规划。
- 位置环输出速度目标。

验收：

- 可以执行有限位移往返。
- 到位后电流不过大。
- 撞限位、跟随误差、堵转能进入 FAULT。

### M7：定点优化和弱磁预留

目标：

- 快环内去除浮点和除法。
- PI、sin/cos、SVPWM 使用定点。
- 预留弱磁参数，但默认关闭。

验收：

- 20 kHz 电流环 ISR 耗时有足够余量。
- 长时间运行无溢出。
- 弱磁关闭时不影响普通运行。

## 13. 下个 AI 接手注意事项

不要做：

- 不要重新加回已经删除的临时调试变量。
- 不要把 MA600 从 `Board` 拆成顶层 `Encoder`。
- 不要把 `App/main.c` 写成大杂烩。
- 不要在快环里阻塞式读写串口或打印。
- 不要为了 C++ 而把 `Motor` 改回 C++。
- 不要在没有电流保护和丝杠限位前长时间开环转动。

优先做：

- 先补 `Motor_FastLoop()` 的快环骨架。
- 再补角度、速度、位置估算。
- 再做 FOC 数学只计算不输出。
- 再小电流、短时间输出。
- 每解决一个调试问题，更新 [[99 学习与排错记录]]。
