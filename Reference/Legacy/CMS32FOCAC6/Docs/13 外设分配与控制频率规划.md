## 时钟规划

### 当前工程实际时钟

`system_CMS32M6510.c` 中 option byte 当前为：

```c
0xEA
```

它对应：

```text
fHOCO = 64 MHz
fIH   = 16 MHz
```

也就是说，当前工程更应该按 **16 MHz 系统时钟** 来起步估算。

注意：Keil `.uvprojx` 里也能看到 `CLOCK(12000000)` 之类的字段，它更像调试/工程元数据，不等于运行时真实 `SystemCoreClock`。运行时以 `SystemCoreClockUpdate()` 和 option byte/HOCODIV 配置为准。

### 推荐阶段

| 阶段 | 系统时钟 | 用途 |
| --- | ---: | --- |
| 第一阶段 | 16 MHz | 空工程、GPIO、MA600 假数据、低速外设验证 |
| 第二阶段 | 16 MHz | 8 kHz PWM/电流环起步，验证 ADC/PGA/EPWM |
| 第三阶段 | 48 MHz 或 64 MHz | 16 kHz 电流环、更多数学计算、较高带宽闭环 |

上板前不要急着改 64 MHz。先用当前 16 MHz 跑通下载、时钟、GPIO、SPI、ADC，再考虑提频。

## SPI/MA600 规划

MA600 读取本质是 16 bit SPI 读角度。安装在空心磁环侧边时，读出的仍是 16 bit 磁场角，但它不一定等于机械角，需要后续做方向、零点和非线性校准。

建议：

| 项目 | 建议 |
| --- | --- |
| SPI 外设 | SSP0 |
| CS 控制 | `P02/CNS` 软件 GPIO 控制 |
| 数据长度 | 16 bit |
| 第一阶段 SCK | 1 MHz |
| 调通后 SCK | 2-4 MHz，以上板波形为准 |
| 初期读取位置 | `Motor_FastLoop()` 中使用假数据 |
| 真实闭环读取位置 | 优先在电流环开始时读取，或者在速度环读取后给电流环使用最近角度 |

SPI 读一次 16 bit 的理论传输时间：

| SCK | 16 bit 传输时间 |
| ---: | ---: |
| 1 MHz | 16 us |
| 2 MHz | 8 us |
| 4 MHz | 4 us |

如果电流环是 8 kHz，周期是 125 us，2 MHz SPI 读取占 8 us，还能接受。如果电流环是 16 kHz，周期是 62.5 us，1 MHz SPI 读取会占掉 25% 以上的周期，不适合阻塞读取。

## ADC/PGA 规划

### 电流采样

本板是三低边电流采样：

| 相 | 采样网络 | PGA 输入 | 可用 ADC |
| --- | --- | --- | --- |
| U | `NU` 通过 R5 80 mOhm | `P00/A0P`, `P01/A0GND` | PGA0 输出或 `AN8` |
| V | `NV` 通过 R6 80 mOhm | `P24/A1P`, `P25/A1GND` | PGA1 输出或 `AN14` |
| W | `NW` 通过 R7 80 mOhm | `P26/A2P`, `P27/A2GND` | PGA2 输出或 `AN16` |

建议第一版使用 PGA 输出采样：

```text
PGA0 -> ADC_CH_0/1  待实测确认具体输出通道
PGA1 -> ADC_CH_2
PGA2 -> ADC_CH_3
NTC  -> ADC_CH_4
VDD  -> ADC_CH_20
GND  -> ADC_CH_21
BG   -> ADC_CH_22
```

板子没到前，文档里不能把 PGA 输出通道写死为最终结论。实际需要用 ADC 示例和万用表/示波器确认。

### ADC 触发结构

推荐结构参考 `01_ELA_ACC`：

```text
EPWM 中心对齐计数
    ↓
EPWM 比较点触发 ADC
    ↓
ADC 完成中断 ADC_IRQn
    ↓
读取三相电流
    ↓
Motor_FastLoop()
    ↓
更新下一周期 PWM 占空比
```

慢速 ADC，如 NTC、VDD、内部 BG，不要放在每次电流环里全部采。推荐轮询：

```text
NTC: 10-100 Hz
VDD: 10-100 Hz
BG/GND: 调试或校准时采
```

## EPWM/PWM 规划

推荐使用 EPWM 中心对齐模式，原因：

- 电流采样点更容易放在 PWM 周期中间。
- 对 FOC 的相电流采样更友好。
- 参考工程也是 PWM 触发 ADC 的结构。

当前 16 MHz 下，若 EPWM 时钟不分频，中心对齐 PWM 的半周期计数近似为：

```text
half_count = Fepwm / (2 * Fpwm)
```

| PWM 频率 | 周期 | 16 MHz half_count | 说明 |
| ---: | ---: | ---: | --- |
| 8 kHz | 125 us | 1000 | 推荐起步值 |
| 10 kHz | 100 us | 800 | 可尝试 |
| 12 kHz | 83.3 us | 667 | 需要注意计算时间 |
| 16 kHz | 62.5 us | 500 | 16 MHz 下偏紧 |
| 20 kHz | 50 us | 400 | 不建议 16 MHz 起步 |

若后续把系统时钟提升到 64 MHz：

| PWM 频率 | 周期 | 64 MHz half_count | 说明 |
| ---: | ---: | ---: | --- |
| 16 kHz | 62.5 us | 2000 | 推荐高性能目标 |
| 20 kHz | 50 us | 1600 | 可评估 |
| 32 kHz | 31.25 us | 1000 | 对 M0+ 和 SPI 角度读取很紧 |

第一版不要在当前 `Board_InitPwm()` 里打开输出。只配置安全状态、mask、inactive、brake，然后等待上板实测。以后如果重命名，可以把它整理成 `Board_PWM_InitSafe()`。

## 中断优先级规划

CMS32M65xx 当前可用中断包括 `EPWM_IRQn`、`ADC_IRQn`、`ACMP_IRQn`、`UART0_IRQn`、`SSP0_IRQn`、`TIMER0_IRQn`、`TIMER1_IRQn`、`LSITIMER_IRQn` 等。

建议逻辑优先级：

| 优先级 | 中断 | 用途 | 要求 |
| ---: | --- | --- | --- |
| 最高 | ACMP/EPWM Brake | 过流、异常保护 | 尽量硬件直接关断，ISR 只记录故障 |
| 高 | ADC_IRQn | 电流采样完成，执行 `Motor_FastLoop()` | 不打印，不阻塞，不做慢任务 |
| 中 | TIMER0_IRQn 或 SysTick | 1 ms 系统节拍 | 设置 flag，少做计算 |
| 中低 | EPWM_IRQn | PWM 状态、故障辅助 | 不做 FOC 主计算 |
| 低 | UART0/SSP0 | 调试通信或非阻塞 SPI | 开发初期可不用 |

在 Cortex-M0+ 上，中断优先级数量有限，不要设计太复杂。关键是保证 ADC 电流环不被低价值任务打断太久。

## 定时器规划

| 定时器/节拍 | 建议用途 |
| --- | --- |
| EPWM | 三相 PWM 和 ADC 触发核心 |
| TIMER0 | 1 ms 系统节拍，驱动当前 `Motor_TASK1ms()` |
| TIMER1 | 后续备用，如性能计时、通信超时、校准计时 |
| SysTick | 初期可给 `delay.c` 使用，不建议长期同时承担复杂调度 |
| TIMER4 | 后续输入捕获、Hall 时序或特殊测量，第一阶段不用 |
| LSITIMER | 低速唤醒/低功耗，当前不用 |

如果使用 `delay.c`，要注意它可能占用 SysTick。正式控制程序更推荐用 TIMER0 产生 1 ms tick，避免 SysTick 和 delay 工具互相抢。

## 三闭环频率估算

### 参考工程的结构

`01_ELA_ACC` 参考工程中：

- PWM 频率参考值有 16 kHz。
- 电流环/FOC 在 ADC 转换完成中断里执行。
- 速度环约 1 kHz。
- 位置、温度、通信、滤波等在更慢任务里运行。

CMS32M6513 是 Cortex-M0+，没有硬件 FPU，当前工程起步时钟还是 16 MHz，所以不能直接照搬 16 kHz 全浮点 FOC。

### 推荐频率表

| 配置 | 电流环 | 速度环 | 位置环 | 说明 |
| --- | ---: | ---: | ---: | --- |
| 16 MHz 保守起步 | 4 kHz | 500 Hz | 100-200 Hz | 适合先用 float、阻塞 SPI、验证流程 |
| 16 MHz 推荐目标 | 8 kHz | 1 kHz | 200 Hz | 适合第一版实用闭环 |
| 16 MHz 上限评估 | 10-12 kHz | 1 kHz | 200-500 Hz | 需要减少 float、避免 ISR 中阻塞打印 |
| 64 MHz 推荐目标 | 16 kHz | 1 kHz | 200-500 Hz | 接近参考工程节拍 |
| 64 MHz 上限评估 | 20 kHz | 1-2 kHz | 500 Hz-1 kHz | 需要实测 ISR 耗时和 SPI 时序 |

### 为什么这样估算

以 16 MHz、8 kHz 电流环为例：

```text
电流环周期 = 1 / 8000 = 125 us
CPU 周期数 = 16 MHz * 125 us = 2000 cycles
```

这 2000 个周期要容纳：

```text
ADC ISR 进入退出
读取三相电流
MA600 角度读取或使用最近角度
角度换算
Clarke/Park
Id/Iq PI
反 Park
SVPWM
写 EPWM 占空比
故障检查
```

如果全部用 float，在 Cortex-M0+ 上会比较吃紧。如果使用定点或轻量整数计算，8 kHz 更现实。

以 16 MHz、16 kHz 电流环为例：

```text
电流环周期 = 62.5 us
CPU 周期数 = 1000 cycles
```

这对 M0+ 来说明显紧，尤其还要读 MA600 SPI。因此 16 MHz 下不建议直接上 16 kHz 电流环。

如果切到 64 MHz、16 kHz：

```text
电流环周期 = 62.5 us
CPU 周期数 = 4000 cycles
```

这才比较接近参考工程的 16 kHz 结构。

## 建议的软件节拍

```text

    Motor_FastLoop()        4-8 kHz 起步

```
