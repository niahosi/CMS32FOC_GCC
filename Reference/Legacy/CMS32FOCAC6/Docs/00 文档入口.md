## 已验证结论

- Keil ARMCC5 构建已通过：`0 Error(s), 0 Warning(s)`。
- 系统时钟已确认是 `64 MHz`。
- EPWM 已能输出三相 `20 kHz` PWM。
- ADC 链路为 EPWM CMP0 触发两相采样，第三相由三相电流和重构。
- MA600 SPI 已能读到角度，ADC 中断内更新角度缓存，快环使用缓存电角度。
- 电机与 MA600 空心磁环都按 `4` 对极处理，FOC 电角度不再额外乘 4。
- 当前正在使用临时 `Motor_ZeroScanTest` 做电角度零点扫描，用于修正 `iq_ref = 0` 时的 dq 偏差。
- 从电机背后看顺时针转动时，MA600 delta 多为负数，当前 `MOTOR_SENSOR_DIR = -1`。
- 当前 `ADC_IRQHandler() -> Board_HandleAdcIrq() -> Motor_FastLoop()` 是快环入口。
- 当前调试配置下电流环约 `10 kHz`（`MOTOR_FAST_LOOP_DIV=2u`），速度环分频配置仍需随电流环频率复核。
- `App_Test`、`Motor_Diag`、旧 `debug_*`、`g_run/g_debug/g_test`、MA600 4 字节诊断入口已从正式源码移除。
- VF/IF 开环仍保留，但已移动到 `Motor_OpenLoop.c/h`，作为相序、电流采样方向和电角度趋势检测工具。
- 位置环只保留文档规划，暂不启用；接回丝杠后必须补限位、回零和跟随误差保护。

## 重要边界

- `App` 只做调度、命令下发和观察输出，不直接配置 ADC、PWM、GPIO。
- `Board` 管本 PCB 的硬件适配和安全默认状态。
- `Config` 只集中调参项，不放寄存器位、引脚复用和外设通道号。
- `Motor` 管电机控制算法、状态机、三闭环和保护策略。
- `Vendor` 尽量不改。
- 后续临时代码必须带 `Test` 或 `Diag`，验证后及时清理。
- 后续教学、排错和关键决策要继续写入 [[99 学习与排错记录]]。
