# 07 DQ坐标偏差诊断

本文记录一个具体问题的完整判断过程：

```text
iq_ref -> PI -> vq 命令链路是通的，
但 dq 坐标没有对正，
也就是“以为的 q 轴”还不是实际纯力矩轴。
```

目标不是只记录结论，而是说明如何从变量异常一步一步判断出这个问题。

## 1. 问题背景

当前工程已经具备这些基础条件：

- MA600 可读角度。
- PWM 可输出。
- ADC/PGA 三相电流采样可用。
- `Motor_FastLoop()` 已能进入闭环骨架。
- 可通过 `debug_motor_enable`、`debug_current_loop_req`、`debug_iq_ref` 做最小闭环测试。

此时最容易犯的错误是：

```text
只看电机转了没转，
或者只看 iq_ref 改了没有，
就开始调 PI。
```

这不够。因为 FOC 是否成立，核心不是“会不会动”，而是：

```text
坐标轴是否对正，
也就是 d/q 分解是否接近物理含义。
```

## 2. 先验知识

先把目标说清楚：

- `d` 轴：理想上对应磁链轴。
- `q` 轴：理想上对应力矩轴。

理想情况下：

```text
给 q 轴电压或 q 轴电流，
应该主要看到 iq 变化，
id 尽量小。
```

如果出现下面这种现象：

```text
给的是 q，
但 id 也很大，
甚至比 iq 还明显，
```

就要怀疑：

- 电角度零点偏了。
- Park 变换角度方向错了。
- 相序和电流相序不一致。
- 电流采样符号或轴定义不一致。

## 3. 第一类现象：命令链路是否通

先验证最基本的问题：

```text
iq_ref 改变后，
vq 会不会跟着变。
```

如果：

```text
iq_ref = +20 -> vq 为正
iq_ref = -20 -> vq 为负
```

说明至少这一段是通的：

```text
iq_ref
-> PI
-> vq
```

这时可以排除：

- 调试变量没写进去。
- `Motor_SetCurrentRef()` 没生效。
- PI 输出完全没工作。

这一步只能说明“命令输出链路通”，不能说明 dq 坐标是对的。

## 4. 第二类现象：闭环瞬时电流不稳定

在当前项目里，闭环测试曾出现：

```text
iq_ref 能改，
vq 能正负变化，
但 iq 本身不稳定。
```

这一步不要急着下结论说“电流环不行”。

先区分两件事：

1. 命令有没有走到输出。
2. 反馈坐标有没有对正。

如果是：

```text
vq 能正负翻转，
iq 却不稳定，
```

说明更像是：

```text
反馈侧的问题还没理顺，
而不是命令侧完全没工作。
```

也就是说，此时不该优先调 PI，而该优先怀疑：

- 角度零点。
- 相位关系。
- 电流坐标映射。

## 5. 第三类现象：+iq_ref 和 -iq_ref 最终都往同一方向跑

项目里还出现过一个很容易误判的现象：

```text
iq_ref = +3 和 iq_ref = -3，
最终 angle_pos 都往同一方向变化。
```

这一现象本身不能直接证明“方向错”。

原因是：

- 当前 `Kp = 1`、`Ki = 0`，只是很弱的 P 环。
- 小幅 `iq_ref` 很容易被反馈偏置、采样噪声和惯性盖住。

因此，当时的正确处理不是立即改相序，而是：

```text
先把测试幅值从 +/-3 提高到 +/-20，
再看 vq 和平均反馈。
```

也就是说：

```text
小给定方向不明显，
不能直接拿来判断坐标轴是否正确。
```

## 6. 关键证据：固定电压 DiagVector

真正把问题指向“dq 没对正”的关键，不是闭环瞬时值，而是固定电压矢量测试。

测试方法：

```text
debug_current_loop_req = 0
使用 DiagVector:
q  : vq = +80
nq : vq = -80
```

然后看这些统计量：

```text
diag_id_avg
diag_iq_avg
diag_vector_pos_delta
```

理想判断逻辑：

```text
q  -> diag_iq_avg 应该明显偏一侧
nq -> diag_iq_avg 应该明显偏另一侧
同时 |diag_id_avg| 应尽量小
```

项目里的实测特点是：

- `q/nq` 时 `diag_iq_avg` 能分正负。
- 但 `diag_id_avg` 仍然不小。

这一步很关键。它说明：

```text
q 轴命令并不是完全没有对应到 iq，
但它同时明显投影到了 id 上。
```

这正是 dq 坐标没有对正的典型信号。

换句话说：

```text
不是“完全不会算 iq”，
而是“q 轴还不够纯”。
```

## 7. 为什么这比“电机会不会转”更可信

只看转子转没转，结论很弱。

原因：

- 电机会受惯性影响。
- 当前是空载自由转，容易滑行。
- 小电流给定时，机械响应可能被摩擦和初始角度影响。
- 即使 dq 轴有偏差，电机也可能仍然会转。

所以判断 dq 是否对正，优先级应该是：

```text
先看 id/iq 的统计投影关系，
再看机械方向。
```

不是反过来。

## 8. 这一问题的标准判断链

如果以后再遇到类似问题，按这个顺序判断：

1. 看命令链路  
   `iq_ref` 改变时，`vq` 是否能正负翻转。

2. 看快环是否真的在跑  
   `fast_count`、`loop_samples` 是否增加。

3. 看闭环平均值，不看单点  
   观察：
   ```text
   loop_iq_avg
   loop_id_avg
   loop_vq_avg
   loop_vd_avg
   ```

4. 看固定电压 `q/nq` 矢量  
   观察：
   ```text
   diag_iq_avg
   diag_id_avg
   ```

5. 得出结论  
   如果：
   ```text
   q/nq 时 iq 能分正负，
   但 id 仍然明显不小，
   ```
   则优先判断为：
   ```text
   dq 坐标未对正，
   应先修零点/相位关系，
   不应先调 PI。
   ```

## 9. 当前项目为何得出这个结论

当前项目已经满足：

- `iq_ref -> vq` 能正负翻转。
- 闭环 `iq` 不稳定，不能稳定跟随 `iq_ref`。
- `DiagVector q/nq` 时 `diag_iq_avg` 能正负分开。
- 但 `diag_id_avg` 仍然不小。

因此当前最核心的工程结论是：

```text
iq_ref -> PI -> vq 这条命令链路是通的。
但 dq 坐标还没有对正，
或者说“以为的 q 轴”还不是实际纯力矩轴。
所以现在不适合调 PI，
应该先修正电角度零点/相位关系。
```

## 10. 下一步正确动作

当前应优先做：

1. 增加在线零点微调量。
2. 固定 `DiagVector q`，扫描零点偏移。
3. 以：
   ```text
   |diag_iq_avg| 尽量大
   |diag_id_avg| 尽量小
   ```
   为判据选零点。
4. 再回到闭环看：
   ```text
   loop_iq_avg
   loop_id_avg
   ```

不应优先做：

- 直接调大 `Kp/Ki`。
- 只看机械方向就改相序。
- 只看瞬时 `iq` 单点数值下结论。

## 11. 最后的判断标准

当下面两条同时满足时，才可以认为 dq 坐标基本对正，可以进入 PI 调整阶段：

1. 固定 `q/nq` 矢量时：
   ```text
   diag_iq_avg 明显大于 diag_id_avg
   ```

2. 闭环 `iq_ref = +X / -X` 时：
   ```text
   loop_iq_avg 能跟着正负变化，
   loop_id_avg 保持较小
   ```

在这之前，PI 参数没有调整意义。

## 12. 严格零点扫描程序

如果 `id_ref = 0`、`iq_ref` 有给定时，实际电流主要跑到 `id` 上，说明电角度零点、相序或电流映射可能没有对正。当前临时零点扫描程序会在每个 `elec_zero_trim` 下自动测试正负 q 轴小电流，并用 `id` 串扰、`iq` 跟随误差和 `vq` 消耗计算分数。

评分思路：

```text
score += abs_avg(id) * 4 + abs_avg(iq_ref - iq) * 4 + abs_avg(vq)
```

分数越小，说明：

- d 轴串扰越小；
- q 轴电流越接近给定；
- 电压消耗没有明显异常。

使用条件：

```text
g_cmd.run.motor_enable = 1
g_cmd.run.ctrl_mode = MOTOR_CTRL_CURRENT
g_cmd.current.id_ref = 0
g_cmd.current.iq_ref = 0   // 扫描启用后由 zero_scan_test 自动接管
```

第一版参数建议：

```text
g_cmd.zero_scan_test.enable = 1
g_cmd.zero_scan_test.start_req = 1
g_cmd.zero_scan_test.iq_ref = 40
g_cmd.zero_scan_test.start_trim = -8192
g_cmd.zero_scan_test.end_trim = 8192
g_cmd.zero_scan_test.step = 512
g_cmd.zero_scan_test.settle_ticks = 100
g_cmd.zero_scan_test.sample_ticks = 200
```

观察：

```text
g_elementary.zero_scan_test.state
g_elementary.zero_scan_test.phase
g_elementary.zero_scan_test.iq_ref
g_elementary.zero_scan_test.trim
g_elementary.zero_scan_test.best_trim
g_elementary.zero_scan_test.score
g_elementary.zero_scan_test.best_score
g_elementary.zero_scan_test.id_avg
g_elementary.zero_scan_test.iq_avg
g_elementary.zero_scan_test.iq_err_avg
g_elementary.zero_scan_test.vq_avg
g_elementary.zero_scan_test.fault
```

其中 `phase = 0` 表示当前在测 `+iq_ref`，`phase = 1` 表示当前在测 `-iq_ref`。

扫描结束后把 `best_trim` 写回：

```text
g_cmd.angle.elec_zero_trim = best_trim
g_cmd.zero_scan_test.enable = 0
```

然后重新测：

```text
id_ref = 0
iq_ref = +40 / -40
```

如果 `id` 明显变小、`iq` 能跟随正负给定，说明零点比原来更接近正确；这时再回到正常电流环 PI 调整。
