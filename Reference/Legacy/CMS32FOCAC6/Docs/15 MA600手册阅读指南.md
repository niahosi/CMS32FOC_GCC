# 15 MA600手册阅读指南

本文教你按页阅读 `D:\wjw\1_Motor\CMS32M65xx\Reference\Hardware\C18221548_磁性角度传感器_MA600GQE-0000-Z_规格书_WJ1591645.PDF`，重点服务当前 `CMS32FOCAC6` 项目的 MA600 SPI 读角度、读寄存器、写寄存器和侧边磁环安装。

## 第 12-13 页：MA600 到底测的是什么

第 12 页讲传感器前端：MA600 检测的是封装中心处的磁场方向，然后输出这个磁场角。

这对你很关键，因为你的 MA600 放在空心磁环侧边，不是放在轴中心。第 13 页明确说明：侧轴安装时，磁场角和机械角不再天然成线性关系。

所以你程序里第一阶段只能把 `MA600_ReadAngle()` 当成“磁场原始角度”，不能直接认为它就是电机机械角。后面要做：

- 方向确认
- 零点偏移
- 磁环极对数确认
- 侧轴非线性校准

## 第 14 页：SPI 模式

第 14 页看两个点：

- MA600 支持 SPI Mode 0 和 Mode 3
- 数据顺序是 MSB first

所以你现在这样配置是合理的：

```c
SSP_ConfigRunMode(SSP_FRAME_SPI, SSP_CPO_0, SSP_CPHA_0, SSP_DAT_LENGTH_8);
```

这里用的是 Mode 0。因为当前 CMS32 SSP 配的是 8 bit，所以读一个 16 bit 角度时，要传两次 8 bit：

```text
发 0x00 -> 收 angle[15:8]
发 0x00 -> 收 angle[7:0]
```

## 第 16 页：SPI Communication

第 16 页是 SPI 通信章节的入口。这里要建立一个概念：

MA600 的 SPI 不只是读角度，还包括：

- Read angle
- Read multi-turn
- Read speed
- Read register
- Write register
- Store to NVM
- Clear error

当前你先实现前三类就够了：

1. 读角度
2. 读寄存器
3. 写寄存器

## 第 20 页：SPI 命令总表

第 20 页是最适合写代码时对照的页面。

你当前代码可以按这张表理解：

### 读角度

手册含义：

```text
发送 1 个 16-bit 0x0000
返回 16-bit angle
```

你的 8 bit SSP 写法：

```c
high = SPI_Transmit(0x00);
low = SPI_Transmit(0x00);
angle = ((uint16_t)high << 8) | low;
```

### 读寄存器

手册含义是两步：

```text
第 1 帧：发送读寄存器命令 + 地址
第 2 帧：发送 dummy，读回寄存器值
```

你现在写成：

```c
SPI_Transmit(0xD2);
SPI_Transmit(addr);

SPI_Transmit(0x00);
value = SPI_Transmit(0x00);
```

这个结构来自手册第 20 页，也和 `acc` 项目的 `Ma600_ReadRegister()` 一致。

### 写寄存器

手册里的写寄存器不是简单发 `addr + value`，它需要先进入写配置流程。`acc` 里已经跑通的写法是：

```text
0xEA, 0x54
addr, value
0x00, 0x00
```

所以你当前 `MA600_SPI_Write()` 也按这个顺序写。

## 第 21 页：带校验的 SPI

第 21 页讲 parity check。你第一阶段可以先不启用 parity。

原因是：

- 当前目标是先确认 SPI 物理链路通
- `acc` 参考工程也先按普通 SPI 命令跑
- parity 会让命令格式更复杂，不适合第一版调试

等读角度稳定后，再考虑是否启用校验。

## 第 24 页：Register Map

第 24 页是寄存器地图。你后面遇到“0x1C 是什么”这种问题，就从这里查。

当前最先关注：

- `0x1C`：`MTSP` 相关，`acc` 用它设置第二个 16 bit 返回 speed 还是 multi-turn
- 零点寄存器
- 方向寄存器
- BCT 相关寄存器
- 错误标志相关寄存器

写代码时不要凭记忆猜地址，先回到第 24 页确认。

## 第 26 页：配置参数总表

第 26 页把重要配置参数集中列出来。你后面会用到：

- Zero setting：零点
- BCT：侧轴安装校准
- Rotation direction：方向
- MTSP：多圈/速度输出选择
- Filter：滤波
- Interface/output：输出接口配置

这页适合和第 24 页一起看：第 26 页告诉你“这个参数是干嘛的”，第 24 页告诉你“它在哪个寄存器”。

## 第 28 页：滤波

第 28 页讲数字滤波。第一版先不要乱改滤波。

建议：

- 先用默认滤波跑通角度读取
- 观察角度跳动和速度估算
- 等闭环需要更高响应或更低噪声时，再回来看滤波设置

## 第 29-31 页：侧轴安装、BCT、用户校准

这是你这块板后面很重要的部分。

第 29 页开始讲侧轴安装时，磁场角与机械角会产生非线性误差。第 29-30 页讲 BCT，第 31 页讲用户校准。

对你来说，顺序应该是：

1. 先读出原始角度
2. 手转磁环，看角度是否连续
3. 确认方向
4. 确认机械角和磁场角的关系
5. 后续再做 BCT 或 32 点校准

不要一开始就写 BCT。你现在还没确认磁环、距离、方向和实际误差。

## 第 34-35 页：速度和多圈

`acc` 初始化里写：

```c
Ma600_WriteRegister(0x1c, 0x80);
```

它的目的就是配置 MA600 第二个 16 bit 返回 speed 相关数据，而不是默认多圈数据。

你在 `CMS32FOCAC6` 里不必第一天就做这个。建议顺序：

1. 先 `MA600_ReadAngle()`
2. 再 `MA600_SPI_Read_SFR(0x1C)`
3. 再 `MA600_SPI_Write(0x1C, 0x80)`
4. 最后做 32 bit 读角度 + 速度

## 第 36-38 页：IO 和 SPI 引脚功能

第 37 页能看到 SPI 输入脚内部上下拉相关内容。第 38 页能看到 IO 功能重映射。

对你当前板子来说，先记住：

- `/CS` 空闲为高
- SCLK、COPI、CIPO 按 SPI 连接
- 不要一开始改 MA600 的 IO 重映射配置

你的板上 `P02/CNS` 接的是 MA600 `/CS`，低电平选中。

## 第 40-41 页：SPI 速查表

这是写代码时最该反复看的地方。

建议你把第 40 页当成 `Board_MA600.c` 的对照页：

- `MA600_ReadAngle()` 对照 Read Angle
- `MA600_SPI_Read_SFR()` 对照 Read Register
- `MA600_SPI_Write()` 对照 Write Register
- 后续 `MA600_ClearError()` 对照 Clear Error Flags
- 后续 `MA600_NVMStore()` 对照 Store to NVM

第一版不要写 NVM 存储。NVM 是永久写入，调试阶段先只写 RAM 配置和回读验证。

## 对照当前代码

当前 `Board_MA600.c` 的核心可以这样对照手册：

| 函数 | 对照页 | 说明 |
|---|---:|---|
| `SPI_Master_Mode()` | 14 | SPI Mode 0，MSB first，8 bit 传输 |
| `SPI_Transmit()` | 14-16 | 单字节全双工收发 |
| `MA600_ReadAngle()` | 20、40 | 两个 dummy 字节读回 16 bit angle |
| `MA600_SPI_Read_SFR()` | 20、40 | 发读命令和地址，再读回寄存器值 |
| `MA600_SPI_Write()` | 20、40 | 先解锁，再写地址和值，再读确认帧 |

## 你的阅读路线

现在阶段按这个节奏看：

1. 写 `MA600_ReadAngle()` 时，看第 14、20、40 页。
2. 写 `MA600_SPI_Read_SFR()` 时，看第 20、24、40 页。
3. 写 `MA600_SPI_Write()` 时，看第 20、24、40 页。
4. 理解为什么侧边磁环读数不等于机械角，看第 12、13、29、30、31 页。
5. 后续要读速度，看第 34、35 页。
