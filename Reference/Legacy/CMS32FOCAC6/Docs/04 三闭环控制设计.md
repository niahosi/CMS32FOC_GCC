# 04 三闭环控制设计

本文记录 `CMS32FOCAC6` 从电流环走向速度环、位置环的当前落地方案。当前优先级仍然是：

```text
电流采样可信化
  -> 电角度零点和 q 轴方向复核
  -> 电流环稳定
  -> 速度估算和速度环
  -> 丝杠位置环
```

当前主调试入口：

```text
g_cmd         用户可写命令
g_elementary 用户只读观察
```

旧 Watch 入口、测试源和独立诊断入口已从正式源码移除。

## 1. 当前基线

```text
编译器      Keil MDK ARMCC5
系统时钟    64 MHz
PWM         20 kHz, PWM_PERIOD = 1600
ADC 触发    EPWM CMP0，默认 tick = PWM_ADC_TRIGGER_TICK_DEFAULT
电流采样    正式闭环建议 CMP0 触发两相 ADC，第三相重构
当前组合    CURRENT_SAMPLE_PAIR = CURRENT_SAMPLE_UVW_DIAG，用于三相采样测试
快环入口    ADC_IRQHandler()
电流环频率  20 kHz / MOTOR_FAST_LOOP_DIV(10) = 2 kHz
速度环频率  2 kHz / MOTOR_SPEED_LOOP_DIV(4) = 500 Hz
```

当前参数来自 `User/Config/Config.h`：

```text
MOTOR_CURRENT_KP = 4
MOTOR_CURRENT_KI = 1
MOTOR_CURRENT_PI_SHIFT = 3
MOTOR_CURRENT_REF_LIMIT = 160
MOTOR_CURRENT_V_LIMIT = 160
MOTOR_SPEED_IQ_LIMIT_DEFAULT = 20
```

这些仍是 bring-up 参数，不代表最终 PI。

## 2. 同步链路

```text
EPWM CMP0
  -> ADC 同步采样
  -> ADC_IRQHandler()
  -> Board_HandleAdcIrq()
  -> Board_UpdateAngleFast()
  -> Motor_UpdateAngleFromIsr()
  -> Motor_FastLoop()
```

设计规则：

- 电流来自 ADC 同步采样。
- MA600 角度在同一个 ADC 中断中更新缓存。
- `Motor_FastLoop()` 不直接发起慢速调试输出。
- 主循环只做命令应用、状态机、快照刷新、RTT 慢输出。

## 3. 电流采样可信化

`Board_Analog` 明确区分三类电流：

```text
raw current:
  ADC/PGA 物理通道扣零漂后的值
  观察：g_elementary.current.iu_raw_cnt / iv_raw_cnt / iw_raw_cnt / raw_sum

logic current:
  给 Motor/FOC 使用的 U/V/W 逻辑相电流
  观察：g_elementary.current.iu / iv / iw / sum

reconstructed current:
  未直接采样的第三相由 Ia + Ib + Ic = 0 重构
```

当前可选组合：

```text
CURRENT_SAMPLE_UV:
  CMP0 触发 ADC_CH_0 / ADC_CH_2
  Iu = rawU, Iv = rawV, Iw = -Iu - Iv

CURRENT_SAMPLE_UW:
  CMP0 触发 ADC_CH_0 / ADC_CH_3
  Iu = rawU, Iw = rawW, Iv = -Iu - Iw

CURRENT_SAMPLE_VW:
  CMP0 触发 ADC_CH_2 / ADC_CH_3
  Iv = rawV, Iw = rawW, Iu = -Iv - Iw
```

判断原则：

- 当前未被 CMP0 触发的 raw 通道会显示为 0 count，这是正常现象。
- `logic sum` 由重构电流得到，不能替代 `raw_sum` 判断硬件三通道一致性。
- 采样明显不可信时，不要调 PI。

## 4. 电角度和 MA600

当前硬件结论：

```text
电机极对数 = 4
MA600 空心磁环 = 4 对极
机械一圈 MA600 原始角变化 4 个周期
FOC 电角度不再额外乘以 4
MOTOR_SENSOR_DIR = -1
```

电角度由 `Motor_GetElecAngle()` 处理方向和零点：

```text
angle_elec = MOTOR_ELEC_ZERO + trim - raw
```

调试零点时写：

```text
g_cmd.angle.elec_zero_trim
```

观察：

```text
g_elementary.angle.raw
g_elementary.angle.elec
g_elementary.angle.pos
g_elementary.angle.delta
```

## 5. 电流环

电流环路径：

```text
g_cmd.current.id_ref / iq_ref
  -> DebugApplyControl()
  -> Motor_SetCurrentRef()
  -> Clarke / Park
  -> PI(id) / PI(iq)
  -> Foc_LimitDq()
  -> InvPark
  -> SVPWM
  -> Board_SetPwmDuty()
```

手动小电流测试：

```text
g_cmd.run.motor_enable = 1
等待 g_elementary.motor.state 进入 CLOSED_LOOP
g_cmd.run.ctrl_mode = MOTOR_CTRL_CURRENT
g_cmd.current.id_ref = 0
g_cmd.current.iq_ref = 0, +1, -1, +2, -2, +3, -3
g_cmd.current.kp = 4, 6, 8...
g_cmd.current.ki = 0, 1...
g_cmd.current.v_limit = 120, 160, 300...
```

观察：

```text
g_elementary.foc.id_ref / iq_ref
g_elementary.foc.id / iq
g_elementary.foc.vd / vq
g_elementary.foc.kp / ki / v_limit
g_elementary.foc.v_limited
g_elementary.pwm.duty_u / duty_v / duty_w
g_elementary.motor.fault_reason
g_elementary.safety.current_over_count
```

判断：

- `vq` 应随 `iq_ref` 正负变化。
- `v_limited` 频繁为 1，说明电压命令顶限，先不要加大给定。
- 正负方向差异很大，优先回查电角度零点、相序、电流采样窗口。

在线调参时，先固定 `ki=0` 只调 `kp`，确认正负小电流稳定后再把 `ki` 从 1 开始加。程序检测到 `kp/ki/v_limit` 改变时会重置电流 PI 积分，避免旧积分带入新参数。

## 6. 开环检测

开环检测保留在 `Motor_OpenLoop.c/h`，不再混在 `Motor_Loop.c` 闭环主线里。

用途：

- VF：用固定电压旋转电角度，检查相序、电压输出、电流采样周期性。
- IF：用开环电角度 + 电流 PI，辅助检查电流方向和采样方向。

入口：

```text
g_cmd.run.ctrl_mode = MOTOR_CTRL_VF
g_cmd.open_loop.ol_speed_ref
g_cmd.open_loop.vf_voltage

g_cmd.run.ctrl_mode = MOTOR_CTRL_IF
g_cmd.open_loop.ol_speed_ref
g_cmd.open_loop.if_id_ref
g_cmd.open_loop.if_iq_ref
```

观察：

```text
g_elementary.open_loop.theta
g_elementary.open_loop.vf_voltage
g_elementary.open_loop.if_id_ref
g_elementary.open_loop.if_iq_ref
g_elementary.current.iu / iv / iw
g_elementary.pwm.duty_u / duty_v / duty_w
```

## 7. 速度环

速度环路径：

```text
g_cmd.speed.speed_ref
  -> 速度 PI
  -> iq_ref
  -> 电流环
```

速度环测试前提：

- 电流环正负小 `iq_ref` 稳定。
- `v_limited` 不长期为 1。
- `g_elementary.speed.fb` 手转方向可解释。

速度环小步测试：

```text
g_cmd.run.motor_enable = 1
等待 g_elementary.motor.state 进入 CLOSED_LOOP
g_cmd.run.ctrl_mode = MOTOR_CTRL_SPEED
g_cmd.speed.iq_limit = 10 到 20
g_cmd.speed.speed_ref = +5000, -5000, 0
```

观察：

```text
g_elementary.speed.ref
g_elementary.speed.fb
g_elementary.foc.iq_ref
g_elementary.foc.iq
g_elementary.foc.vq
g_elementary.motor.fault_reason
```

## 8. 位置环规划

位置环当前不启用。后续接回丝杠后再做：

```text
position_ref
  -> 轨迹规划/限位
  -> position loop
  -> speed_ref
  -> speed loop
  -> iq_ref
  -> current loop
  -> PWM
```

位置环前必须补齐：

- 机械零点或回零流程。
- 正负软限位。
- 最大速度和最大加速度。
- 跟随误差保护。
- 堵转/过载保护。
- 丝杠导程和实际位移换算。

## 9. 三闭环里程碑

```text
M1 电流采样可信化
M2 电角度零点和 q 轴方向复核
M3 电流环 PI 整定
M4 速度估算稳定化
M5 速度环小速度闭环
M6 丝杠位置环、限位、回零
M7 资源优化和快环频率提升
```

当前不要直接做位置环。只要电流采样或电流环仍明显抖动，速度环和位置环都会把问题放大。
