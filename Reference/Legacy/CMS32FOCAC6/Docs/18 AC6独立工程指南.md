# 18 AC6独立工程指南

本文记录如何在 Keil MDK 中创建 `CMS32M6513AGE40NB` 的 AC6 独立工程，以及各类必备文件的作用。

## 目标

工程不依赖 RTE 自动添加 `Device -> Startup`，而是手动加入启动文件、系统文件、CMSIS、厂商驱动和自己的应用代码。

当前芯片为 `CMS32M6513AGE40NB`，但原厂 pack 内的 Device 文件命名为 `CMS32M6510`。这是原厂 pack 的家族共用命名，不代表芯片选错。

## Keil 中建议的工程分组

在 Keil 左侧工程树中建议建这些 Group：

```text
App
Board
Motor
Vendor_Device
Vendor_Startup
Vendor_Driver
```

不要把所有文件都放进 `Source Group 1`。

## 第一版必须添加的源文件

```text
User/App/Src/main.c
User/Board/Src/Board.c
User/Motor/Src/Motor.c

Vendor/Cmsemicon/CMS32M65xx/Device/Source/system_CMS32M6510.c
Vendor/Cmsemicon/CMS32M65xx/Device/Source/ARM/startup_CMS32M6510.s

Vendor/Cmsemicon/CMS32M65xx/Driver/src/cgc.c
Vendor/Cmsemicon/CMS32M65xx/Driver/src/gpio.c
Vendor/Cmsemicon/CMS32M65xx/Driver/src/delay.c
Vendor/Cmsemicon/CMS32M65xx/Driver/src/ssp.c
```

只加入 `startup_CMS32M6510.s`，不要同时加入 `startup_CMS32M6510.c`。

也不要加入：

```text
Vendor/Cmsemicon/CMS32M65xx/Device/Source/GCC/startup_CMS32M6510.S
Vendor/Cmsemicon/CMS32M65xx/Device/Source/IAR/startup_CMS32M6510.s
```

它们不是 Keil AC6/ARMCC 工程要用的启动文件。

## 第一版必须添加的头文件目录

`Options for Target -> C/C++ -> Include Paths` 中添加：

```text
.\User\App\Inc
.\User\Board\Inc
.\Vendor\Cmsemicon\CMS32M65xx\CMSIS\Include
.\Vendor\Cmsemicon\CMS32M65xx\Device\Include
.\Vendor\Cmsemicon\CMS32M65xx\Driver\inc
.\Vendor\ARM\CMSIS\Core\Include
```

如果没有把 ARM CMSIS 复制进工程，也可以临时使用 pack 路径：

```text
C:\Users\EDY\AppData\Local\Arm\Packs\ARM\CMSIS\5.9.0\CMSIS\Core\Include
```

缺少 `Vendor/ARM/CMSIS/Core/Include` 时，常见错误是：

```text
core_cm0plus.h: error: 'cmsis_version.h' file not found
```

## 各文件作用

### startup_CMS32M6510.s

启动文件，负责：

- 定义中断向量表
- 定义复位入口 `Reset_Handler`
- 设置初始栈
- 上电后调用 `SystemInit()`
- 跳转到 C 运行库入口，最终进入 `main()`

`CMS32M6513AGE40NB` 在该 pack 中使用同一套 `startup_CMS32M6510.s`。

### system_CMS32M6510.c

系统文件，负责：

- 提供 `SystemInit()`
- 提供 `SystemCoreClockUpdate()`
- 定义 `SystemCoreClock`
- 处理系统时钟相关变量

### CMS32M6510.h

芯片寄存器头文件，负责：

- 定义外设寄存器结构
- 定义中断号
- 定义寄存器地址和 bit 位
- 包含 Cortex-M0+ 内核头文件 `core_cm0plus.h`

### core_cm0plus.h 与 ARM CMSIS Core

ARM Cortex-M0+ 内核头文件，负责：

- NVIC
- SysTick
- SCB
- `__enable_irq()`
- `__disable_irq()`
- `SysTick_Config()`
- `NVIC_EnableIRQ()`

它管 ARM 内核，不管 CMS32 外设。

### common.h

厂商驱动公共头文件。多数驱动通过它间接包含 `CMS32M6510.h`。

### cgc.c / cgc.h

时钟和复位相关驱动，负责：

- 外设时钟开关
- 复位原因读取
- 时钟控制相关寄存器操作

### gpio.c / gpio.h

GPIO 驱动，负责：

- IO 输入输出配置
- 引脚复用配置
- GPIO 电平读写

### delay.c / delay.h

延时驱动，负责：

- `m0_delay_ms()`
- `m0_delay_us()`
- `delay_init()`

该驱动使用 SysTick，后续如果自己做系统 tick，要避免冲突。

### ssp.c / ssp.h

SPI/SSP 驱动，负责：

- SPI 主机配置
- SPI 发送接收
- 后续读取 MA600 原始角度

### User/App/Src/main.c

用户程序入口，负责：

- 调用系统/板级初始化
- 进入主循环
- 后续运行应用状态机

### User/Board/Src/Board.c

板级文件，负责：

- 按本项目原理图配置引脚
- `P02/CNS` 默认拉高
- 保留 `P06/P07` SWD
- 配置 MA600 SPI 引脚
- 后续配置 ADC、PGA、EPWM 和保护

## 第一版暂时不要添加的文件

```text
flash.c
adc.c
pga.c
epwm.c
acmp.c
uart.c
i2c.c
wdt.c
crc.c
dac.c
timer4.c
lsitimer.c
```

建议按需求逐步添加。尤其 `flash.c` 涉及运行时擦写 Flash，并使用 `ram_fetch_code` 段；当前不做 bootloader，也不做参数保存时先不要加。

## 常见错误

### 1. 重复定义中断向量

错误示例：

```text
Symbol __Vectors multiply defined
Symbol SysTick_Handler multiply defined
```

原因：同时加入了 `startup_CMS32M6510.s` 和 `startup_CMS32M6510.c`。

处理：只保留 `startup_CMS32M6510.s`。

### 2. 加错 GCC 启动文件

错误示例：

```text
startup_CMS32M6510.S: invalid instruction
conditional execution not supported in Thumb1
```

原因：加入了 `Device/Source/GCC/startup_CMS32M6510.S`。

处理：移除 GCC 启动文件，只使用 `Device/Source/ARM/startup_CMS32M6510.s`。

### 3. 找不到 cmsis_version.h

错误示例：

```text
core_cm0plus.h: error: 'cmsis_version.h' file not found
```

原因：只复制了 `core_cm0plus.h`，没有复制完整 ARM CMSIS Core Include。

处理：复制或引用：

```text
C:\Users\EDY\AppData\Local\Arm\Packs\ARM\CMSIS\5.9.0\CMSIS\Core\Include
```

## 下一步建议

1. 下载确认能进入 `main()`。
2. 在 `Board.c` 中配置 `P02/CNS` 为 GPIO 输出高电平。
3. 添加 `ssp.c` 后，先读 MA600 原始 16 bit 角度。
4. MA600 稳定后，再添加 ADC/PGA。
5. 最后再添加 EPWM/ACMP。
