# 12 外设开发流程

本文记录 MCU/DSP 外设开发的通用思路，并结合本项目 `CMS32FOCAC6`、STM32、ESP32、TI C2000/DSP 这几类平台说明。目标是建立一套稳定的阅读例程和初始化外设的方法，而不是只记某一个芯片的函数名。

## 总结

不同芯片的外设初始化 API 不一样，但基本流程高度相似：

```text
1. 确认硬件连接和引脚复用
2. 配置系统时钟或确认当前主频
3. 打开外设模块时钟
4. 配置 GPIO 模式或复用功能
5. 复位/清空外设状态
6. 配置外设工作模式
7. 配置外设速度、分频、周期、采样时间
8. 配置数据通道、触发源、DMA 或中断
9. 清中断标志
10. 配置 NVIC/PIE/中断控制器
11. 使能外设
12. 启动一次转换、传输、计数或输出
13. 用最小测试验证
```

很多初学者容易混淆这几个概念：

```text
打开外设时钟 != 外设开始工作
配置 GPIO 复用 != 外设开始工作
使能外设 != 输出已经打开
打开中断 != 中断一定会进
启动 PWM 计数 != 功率输出已经安全
```

对电机控制板来说，还要特别加一条：

```text
PWM 和功率输出必须最后打开，且默认保持关闭或刹车状态。
```

## 为什么所有芯片都差不多

MCU 外设本质上都是一组寄存器。无论是 CMS32、STM32、ESP32 还是 C2000，都绕不开这些问题：

- 外设寄存器有没有时钟。
- 引脚是不是切到了外设功能。
- 外设运行模式是什么。
- 外设时钟从哪里来，分频是多少。
- 数据从哪个通道进来，往哪个通道出去。
- 事件由谁触发。
- 中断标志怎么清。
- 什么时候真正开始运行。

所以不同平台只是在函数名字和配置结构体上不同。

## 平台对比

| 平台 | 常见初始化风格 | 外设时钟 | 引脚复用 | 中断 | 特点 |
| --- | --- | --- | --- | --- | --- |
| CMS32M65xx | 厂商驱动函数 + 直接寄存器 | `CGC_PERxxPeriphClockCmd()` | `GPIO_PinAFOutConfig()` | `NVIC_EnableIRQ()` | 小 MCU，外设更贴近寄存器 |
| STM32 HAL | `MX_xxx_Init()` + HAL 结构体 | `__HAL_RCC_xxx_CLK_ENABLE()` | `HAL_GPIO_Init()` Alternate | `HAL_NVIC_EnableIRQ()` | CubeMX 自动生成多 |
| STM32 LL | LL 函数 + 寄存器风格 | `LL_APB...EnableClock()` | `LL_GPIO_Init()` | `NVIC_EnableIRQ()` | 比 HAL 轻，适合电机控制 |
| ESP32 ESP-IDF | driver config 结构体 | driver 内部或 `periph_module_enable()` | GPIO Matrix / IO_MUX | ISR service / FreeRTOS | 外设经常和 FreeRTOS/driver 绑定 |
| TI C2000/DSP | DriverLib + 寄存器 | `SysCtl_enablePeripheral()` | `GPIO_setPinConfig()` | PIE + CPU int | 电机控制外设强，ePWM/ADC/SOC 很核心 |

## 通用阅读例程方法

拿到一个外设例程，不要先纠结每个寄存器 bit，先按这个顺序圈出来：

```text
1. 它开了哪些时钟
2. 它用了哪些引脚
3. 它设置了什么模式
4. 它设置了什么速度或周期
5. 它启用了哪些通道
6. 它用了软件触发还是硬件触发
7. 它有没有中断或 DMA
8. 它最后在哪里真正 Start/Enable
9. 它的中断函数在哪里
10. 哪些代码不能照搬到自己的板子
```

对本项目尤其要找出这些危险点：

```text
EPWM_EnableOutput()
EPWM_Start()
EPWM_DisableSoftwareBrake()
把 P06/P07 改成 UART
把 P00/P24/P26 改成数字 BKIN
在 ADC/EPWM 中断里 printf
```

## GPIO 开发流程

GPIO 是所有外设的地基。

通用流程：

```text
1. 查原理图，确认引脚接了什么
2. 查数据手册，确认这个引脚支持什么功能
3. 打开 GPIO/PORT 时钟
4. 选择普通 GPIO 或复用功能
5. 设置输入、输出、开漏、上拉、下拉、模拟
6. 设置输出初始电平
7. 读取或翻转测试
```

CMS32 示例：

```c
GPIO_PinAFOutConfig(P03CFG, IO_OUTCFG_P03_SCK);
GPIO_Init(PORT0, PIN3, OUTPUT);
```

STM32 HAL 类似：

```c
__HAL_RCC_GPIOA_CLK_ENABLE();

GPIO_InitTypeDef GPIO_InitStruct = {0};
GPIO_InitStruct.Pin = GPIO_PIN_5;
GPIO_InitStruct.Mode = GPIO_MODE_AF_PP;
GPIO_InitStruct.Pull = GPIO_NOPULL;
GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_HIGH;
GPIO_InitStruct.Alternate = GPIO_AF5_SPI1;
HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);
```

ESP32 类似：

```c
gpio_config_t io_conf = {
    .pin_bit_mask = 1ULL << GPIO_NUM_5,
    .mode = GPIO_MODE_OUTPUT,
    .pull_up_en = GPIO_PULLUP_DISABLE,
    .pull_down_en = GPIO_PULLDOWN_DISABLE,
    .intr_type = GPIO_INTR_DISABLE,
};
gpio_config(&io_conf);
```

C2000 类似：

```c
GPIO_setPinConfig(GPIO_0_EPWM1A);
GPIO_setDirectionMode(0, GPIO_DIR_MODE_OUT);
GPIO_setPadConfig(0, GPIO_PIN_TYPE_STD);
```

## SPI 开发流程

SPI 外设通常用于传感器、Flash、编码器。你本板的 MA600 就是 SPI。

通用流程：

```text
1. 查从设备手册，确认 CPOL/CPHA、位宽、最大 SCK、片选时序
2. 打开 SPI 外设时钟
3. 配置 SCK/MISO/MOSI/CS 引脚
4. 设置主机或从机模式
5. 设置 CPOL/CPHA
6. 设置数据位宽，例如 8 bit/16 bit
7. 设置 SPI 分频，得到目标 SCK
8. 配置 CS 是硬件控制还是软件 GPIO
9. 清 FIFO 或状态标志
10. 使能 SPI
11. 拉低 CS，发送 dummy 或命令，等待接收完成，拉高 CS
```

CMS32 参考：

```c
CGC_PER12PeriphClockCmd(CGC_PER12Periph_SPI, ENABLE);
SSP_ConfigClk(7, 2);
SSP_ConfigRunMode(SSP_FRAME_SPI, SSP_CPO_0, SSP_CPHA_0, SSP_DAT_LENGTH_16);
SSP_EnableMasterMode();
SSP_DisableMasterAutoControlCS();
SSP_Start();
```

CMS32 SPI 分频公式：

```text
SSPCLK = PCLK / ((M + 1) * N)
```

例如当前 16 MHz：

```text
M = 7, N = 2
SSPCLK = 16 MHz / ((7 + 1) * 2) = 1 MHz
```

STM32 HAL 通常是：

```c
hspi1.Instance = SPI1;
hspi1.Init.Mode = SPI_MODE_MASTER;
hspi1.Init.Direction = SPI_DIRECTION_2LINES;
hspi1.Init.DataSize = SPI_DATASIZE_16BIT;
hspi1.Init.CLKPolarity = SPI_POLARITY_LOW;
hspi1.Init.CLKPhase = SPI_PHASE_1EDGE;
hspi1.Init.NSS = SPI_NSS_SOFT;
hspi1.Init.BaudRatePrescaler = SPI_BAUDRATEPRESCALER_16;
HAL_SPI_Init(&hspi1);
```

ESP32 ESP-IDF 通常是：

```c
spi_bus_config_t buscfg = {
    .miso_io_num = MISO_PIN,
    .mosi_io_num = MOSI_PIN,
    .sclk_io_num = SCLK_PIN,
};
spi_bus_initialize(SPI2_HOST, &buscfg, SPI_DMA_DISABLED);

spi_device_interface_config_t devcfg = {
    .clock_speed_hz = 1 * 1000 * 1000,
    .mode = 0,
    .spics_io_num = CS_PIN,
    .queue_size = 1,
};
spi_bus_add_device(SPI2_HOST, &devcfg, &spi);
```

C2000 通常是：

```c
SysCtl_enablePeripheral(SYSCTL_PERIPH_CLK_SPIA);
GPIO_setPinConfig(GPIO_16_SPIA_SIMO);
GPIO_setPinConfig(GPIO_17_SPIA_SOMI);
GPIO_setPinConfig(GPIO_18_SPIA_CLK);
SPI_setConfig(SPIA_BASE, DEVICE_LSPCLK_FREQ, SPI_PROT_POL0PHA0,
              SPI_MODE_MASTER, 1000000, 16);
SPI_enableModule(SPIA_BASE);
```

## UART 开发流程

UART 用于调试、通信、上位机协议。

通用流程：

```text
1. 查引脚是否和调试口复用
2. 打开 UART 时钟
3. 配置 TX/RX 引脚复用
4. 设置波特率
5. 设置数据位、停止位、校验位
6. 选择轮询、中断或 DMA
7. 清标志
8. 使能 UART
9. 发送一个字节测试
```

本项目注意：

```text
P06/P07 与 SWD 共用，开发阶段不要急着切 UART。
```

STM32 常见结构：

```c
huart1.Instance = USART1;
huart1.Init.BaudRate = 115200;
huart1.Init.WordLength = UART_WORDLENGTH_8B;
huart1.Init.StopBits = UART_STOPBITS_1;
huart1.Init.Parity = UART_PARITY_NONE;
huart1.Init.Mode = UART_MODE_TX_RX;
HAL_UART_Init(&huart1);
```

ESP32 常见结构：

```c
uart_config_t uart_config = {
    .baud_rate = 115200,
    .data_bits = UART_DATA_8_BITS,
    .parity = UART_PARITY_DISABLE,
    .stop_bits = UART_STOP_BITS_1,
};
uart_param_config(UART_NUM_1, &uart_config);
uart_set_pin(UART_NUM_1, TX_PIN, RX_PIN, UART_PIN_NO_CHANGE, UART_PIN_NO_CHANGE);
uart_driver_install(UART_NUM_1, 1024, 1024, 0, NULL, 0);
```

## ADC 开发流程

ADC 最容易出错，因为它不仅是“打开 ADC”，还涉及参考源、采样时间、触发源、通道顺序、模拟前端。

通用流程：

```text
1. 查模拟输入来源，确认电压范围
2. 配 GPIO 为模拟模式
3. 打开 ADC 时钟
4. 打开参考源或模拟 LDO
5. 配置 ADC 分频
6. 配置采样时间
7. 配置参考电压
8. 配置通道
9. 选择软件触发或硬件触发
10. 配置中断或 DMA
11. 校准 ADC，如芯片支持
12. 启动转换
13. 读取结果并换算
```

CMS32 软件触发 ADC 大致是：

```c
CGC_PER13PeriphClockCmd(CGC_PER13Periph_ADCLDO, ENABLE);
ADCLDO_OutVlotageSel(ADCLDO_OutV_3d6);
ADCLDO_Enable();

CGC_PER13PeriphClockCmd(CGC_PER13Periph_ADCEN, ENABLE);
ADC_ConfigRunMode(ADC_MODE_HIGH, ADC_CONVERT_SINGLE, ADC_CLK_DIV_32, 100);
ADC_ConfigChannelSwitchMode(ADC_SWITCH_HARDWARE);
ADC_DisableChargeAndDischarge();
ADC_ConfigVREF(ADC_VREFP_AVREFP);
ADC_EnableScanChannel(ADC_CH_4_MSK);
ADC_Start();
```

电机电流采样要比普通 ADC 更复杂：

```text
EPWM 比较点触发 ADC
ADC 完成中断
读取电流
扣零漂
调用电流环
更新 PWM
```

STM32 常见有两种：

```text
规则组 ADC：慢速采样，例如温度、电压
注入组 ADC：电机电流，常由 TIM1/TIM8 触发
```

ESP32 ADC 通常更适合慢速采样，不适合非常严格的电流环同步采样。

C2000 常用结构：

```text
ePWM SOCA/SOCB 触发 ADC SOC
ADC EOC 触发中断
中断里执行电流环
```

## PGA/运放开发流程

PGA 是模拟前端，不是所有 MCU 都有。CMS32M65xx 有 PGA0/1/2，很适合低边电流采样。

通用流程：

```text
1. 查采样电阻和输入范围
2. 确认 PGA 输入脚和参考脚
3. 打开 PGA 时钟
4. 配 GPIO 为模拟输入
5. 配置参考电压
6. 配置单端/差分模式
7. 配置增益
8. 使能输出
9. 启动 PGA
10. ADC 读取 PGA 输出
```

CMS32 示例：

```c
CGC_PER13PeriphClockCmd(CGC_PER13Periph_PGA0EN, ENABLE);
PGA_VrefCtrl(PGA0x, VrefHalf);
PGA_EnableOutput(PGA0x);
PGA_ConfigGain(PGA0x, PGA_GAIN_2);
PGA_ModeSet(PGA0x, PgaDiffer);
PGA_Start(PGA0x);
```

对本板来说：

```text
PGA0: P00/P01
PGA1: P24/P25
PGA2: P26/P27
```

上板后先用低增益，确认零点和最大电流，再调增益。

## PWM/EPWM 开发流程

PWM 在电机控制里最危险，因为它直接关系到功率输出。

通用流程：

```text
1. 查功率级结构
2. 打开 PWM 外设时钟
3. 配置 PWM 引脚或内部驱动映射
4. 配置计数模式，建议电机用中心对齐
5. 配置周期
6. 配置初始占空比
7. 配置互补输出
8. 配置死区
9. 配置输出极性
10. 配置加载时机
11. 配置刹车/保护
12. 配置 ADC 触发点
13. 清故障标志
14. 启动计数
15. 最后才使能输出
```

CMS32 参考：

```c
CGC_PER11PeriphClockCmd(CGC_PER11Periph_EPWM, ENABLE);

EPWM_ConfigRunMode(EPWM_COUNT_UP_DOWN |
                   EPWM_OCU_SYMMETRIC |
                   EPWM_WFG_COMPLEMENTARYK |
                   EPWM_OC_INDEPENDENT);

EPWM_ConfigChannelClk(EPWM0, EPWM_CLK_DIV_1);
EPWM_ConfigChannelPeriod(EPWM0, period);
EPWM_ConfigChannelSymDuty(EPWM0, duty);
```

本项目安全规则：

```text
可以配置 EPWM。
可以配置周期和占空比。
可以启动计数用于 ADC 触发验证。
但不要在初始化里直接 EnableOutput。
```

危险函数要谨慎：

```text
EPWM_EnableOutput()
EPWM_Start()
EPWM_DisableSoftwareBrake()
```

STM32 电机 PWM 通常用 TIM1/TIM8：

```text
TIM1 center-aligned
CH1/1N, CH2/2N, CH3/3N
deadtime
break input
TRGO/CC event trigger ADC
```

C2000 通常用 ePWM：

```text
ePWM time-base
CMPA/CMPB
AQ action qualifier
DB dead-band
TZ trip-zone
SOCA trigger ADC
```

ESP32 典型使用 MCPWM 或 LEDC。电机控制更推荐 MCPWM：

```text
mcpwm_timer
mcpwm_operator
mcpwm_generator
deadtime
fault/brake
sync
capture
```

## 定时器开发流程

定时器可以做 1 ms tick、超时、输入捕获、编码器、触发 ADC。

通用流程：

```text
1. 打开定时器时钟
2. 选择时钟源
3. 设置分频
4. 设置计数模式
5. 设置周期
6. 清标志
7. 开中断
8. 配 NVIC
9. 启动定时器
```

CMS32 TIMER0 1 ms 示例思路：

```c
CGC_PER11PeriphClockCmd(CGC_PER11Periph_TIMER01, ENABLE);
TMR_ConfigRunMode(TMR0, TMR_COUNT_PERIOD_MODE, TMR_BIT_32_MODE);
TMR_ConfigClk(TMR0, TMR_CLK_DIV_1);
TMR_SetPeriod(TMR0, SystemCoreClock / 1000 - 1);
TMR_ClearOverflowIntFlag(TMR0);
TMR_EnableOverflowInt(TMR0);
NVIC_SetPriority(TIMER0_IRQn, 2);
NVIC_EnableIRQ(TIMER0_IRQn);
TMR_Start(TMR0);
```

中断里最好只做短任务：

```c
void TIMER0_IRQHandler(void)
{
    if (TMR_GetOverflowIntFlag(TMR0))
    {
        TMR_ClearOverflowIntFlag(TMR0);
        g_tick_1ms = 1;
    }
}
```

## 中断开发流程

通用流程：

```text
1. 外设内部允许产生中断
2. 清除外设中断标志
3. 配置中断优先级
4. 在中断控制器中打开 IRQ
5. 写正确名字的 IRQHandler
6. 在 ISR 里判断来源
7. 清标志
8. 执行极短处理
```

CMS32 使用 NVIC：

```c
NVIC_SetPriority(ADC_IRQn, 1);
NVIC_EnableIRQ(ADC_IRQn);
```

STM32 也是 NVIC：

```c
HAL_NVIC_SetPriority(ADC1_2_IRQn, 1, 0);
HAL_NVIC_EnableIRQ(ADC1_2_IRQn);
```

C2000 是 PIE + CPU interrupt：

```text
注册 PIE 向量
PIE enable
CPU enable group interrupt
EINT
```

ESP32 通常用 driver 注册 ISR 或 event queue，很多驱动会放进 FreeRTOS 任务模型里。

中断里不要做：

```text
printf
阻塞式 SPI/UART
malloc/free
复杂浮点打印
长时间 while 等待
Flash 擦写
```

## DMA 开发流程

DMA 不是必须，但在高速 ADC、UART、SPI 场景很有用。

通用流程：

```text
1. 打开 DMA 时钟
2. 配外设地址
3. 配内存地址
4. 配传输方向
5. 配数据宽度
6. 配传输长度
7. 配循环或单次模式
8. 配 DMA 中断
9. 把外设 DMA 请求打开
10. 启动 DMA
11. 启动外设
```

电机控制里常见：

```text
ADC + DMA 采慢速通道
UART + DMA 做调试输出
SPI + DMA 读大量传感器数据
```

但本项目第一阶段先不用 DMA，先把轮询和中断跑通。

## 外设开发的推荐顺序

对本项目建议：

```text
1. GPIO
2. SPI + MA600 假读/真读
3. TIMER0 1 ms tick
4. ADC 读 NTC
5. ADCLDO + PGA + ADC 读电流零点
6. EPWM 安全初始化，输出关闭
7. EPWM 触发 ADC
8. ADC 中断中跑 Motor_FastLoop()
9. 示波器确认 PWM
10. 限流 VM 下低占空比开环
11. 闭环 FOC
```

不要反过来先写完整 FOC。没有验证外设之前，FOC 只是悬空算法。

## 每个外设的最小验证

| 外设 | 最小验证 |
| --- | --- |
| GPIO | 引脚能输出高低，输入能读到状态 |
| SPI | 逻辑分析仪看到 SCK/MOSI/CS，MISO 有返回 |
| MA600 | 原始角度手转会连续变化 |
| ADC | 读 GND 接近 0，读 VDD/分压符合预期 |
| PGA | 无电流时零点稳定，有小电流时方向正确 |
| EPWM | 示波器看到频率、占空比、互补、死区正确 |
| TIMER | 1 ms tick 频率正确 |
| 中断 | 只进预期中断，标志能清掉 |
| UART | 能发固定字符串，不影响 SWD |

## 一句话记忆

```text
先给外设时钟，再给外设引脚，再给外设模式，再给外设速度，再给外设通道，再给中断，最后才启动。
```

电机控制再加一句：

```text
先保护，后输出；先采样，后闭环；先示波器，后接电机。
```
