# 03 代码结构分析

> 更新日期：2026-06-10  
> 状态：正式 FOC 主线清理后版本  
> 编译器：Keil ARMCC5

本文用于说明当前 `CMS32FOCAC6` 程序结构、运行流程和后续维护边界。当前源码已删除常驻测试源 `App_Test`、独立诊断模块 `Motor_Diag`、旧 `debug_*` 变量、`g_run/g_debug/g_test` 观察表和 MA600 4 字节诊断入口。VF/IF 开环保留，但已独立到 `Motor_OpenLoop.c/h`。

## 1. 总体结构

```text
User/App
  main.c
  App_Debug.c/h

User/Board
  Board.c/h
  Board_Analog.c/h
  Board_MA600.c/h
  Board_PWM.c/h

User/Config
  Config.h

User/Motor
  Motor.c/h
  Motor_Loop.c/h
  Motor_OpenLoop.c/h
  Motor_Foc.c/h
```

职责边界：

- `App`：初始化后的主循环调度、调试命令下发、观察快照刷新、RTT 慢输出。
- `Board`：本 PCB 的外设初始化和硬件适配，包括 MA600、ADC/PGA、EPWM、驱动使能。
- `Motor`：状态机、传感器安全检查、闭环控制、开环检测、FOC 数学和保护策略。
- `Config`：可调参数入口，不放寄存器细节和引脚复用。

## 2. 启动流程

`main()` 当前流程：

```text
Board_Init()
  -> InitClock()
  -> InitGpio()
  -> InitMa600Spi()
  -> Board_UpdateAngle()
  -> Board_InitAdc()
  -> Board_InitPwm()
  -> Board_CalibCurrentOffset()
  -> Board_InitPwmAdcSync()

Motor_Init()
Debug_InitRtt()

while (1)
  -> DebugApplyControl()
  -> Motor_TASK()
  -> Debug_UpdateElementary()
  -> Debug_RttTask()
  -> m0_delay_us(200)
```

设计含义：

- 板级外设先进入安全状态，再启动电机控制状态机。
- 电流零漂在 `Board_Init()` 内完成，PWM 触发 ADC 同步采样也在板级初始化末尾打开。
- 主循环不直接操作寄存器，不直接做 FOC 计算。

## 3. 中断与快环

ADC 中断是当前快环入口：

```text
ADC_IRQHandler()
  -> Board_HandleAdcIrq()
  -> Board_UpdateAngleFast()
  -> Motor_UpdateAngleFromIsr()
  -> if g_cmd.run.motor_enable != 0: Motor_FastLoop()
```

当前频率关系：

```text
PWM/ADC sync = 20 kHz
Motor_FastLoop 分频 = MOTOR_FAST_LOOP_DIV = 10
电流环约 2 kHz
速度环约 500 Hz
```

中断路径禁止加入阻塞打印、动态内存和长耗时诊断。MA600 快速读角已经在中断内执行，因此后续如果提高快环频率，必须重新评估 SPI 读角耗时。

## 4. Board 层

### Board.c

`Board.c` 是板级统一入口，负责把具体外设组合成项目需要的板级能力：

```text
Board_Init()
Board_HandleAdcIrq()
Board_UpdateAngle()
Board_UpdateAngleFast()
Board_GetAngleRaw()
Board_IsAngleOk()
Board_GetAngleAge()
Board_ReadAngleRaw()
```

当前已删除 MA600 4 字节诊断桥接函数，Board 层只保留正式角度缓存和原始角度读取。

### Board_Analog.c

当前运行时电流采样是“两相 ADC + 第三相重构”：

```text
CURRENT_SAMPLE_UV: Iu = rawU, Iv = rawV, Iw = -Iu - Iv
CURRENT_SAMPLE_UW: Iu = rawU, Iw = rawW, Iv = -Iu - Iw
CURRENT_SAMPLE_VW: Iv = rawV, Iw = rawW, Iu = -Iv - Iw
```

当前配置在 `Config.h`：

```text
CURRENT_SAMPLE_PAIR = CURRENT_SAMPLE_VW
PWM_ADC_TRIGGER_TICK_DEFAULT = 650
```

观察时要区分：

- `raw_cnt`：物理 ADC/PGA 通道扣零漂后的值。
- `current.iu/iv/iw`：FOC 当前使用的运行电流。
- `sum`：重构后的三相和，天然接近 0，不能当作三物理通道一致性判断。

### Board_PWM.c

`Board_PWM.c` 负责：

- EPWM 20 kHz 中心对齐互补输出。
- 死区配置。
- 驱动使能脚控制。
- 软件刹车和输出 mask。
- CMP0 ADC 触发点配置。

安全默认状态是 `Board_ForcePwmOff()`：

```text
驱动使能关闭
软件刹车打开
EPWM 输出关闭
占空比回到 50%
```

### Board_MA600.c

`Board_MA600.c` 当前保留：

- SPI 初始化。
- MA600 寄存器基础读写。
- 2 字节角度读取。
- 阻塞式角度缓存更新。
- 中断用快速角度缓存更新。

已删除：

- 4 字节诊断读取缓存。
- speed 诊断寄存器配置入口。
- `Diag` getter。

## 5. Motor 状态机

`Motor.c` 是电机生命周期控制入口：

```text
MOTOR_STATE_IDLE
  -> MOTOR_STATE_SENSOR_CHECK
  -> MOTOR_STATE_ALIGN
  -> MOTOR_STATE_CLOSED_LOOP
  -> MOTOR_STATE_FAULT
```

关键逻辑：

- `IDLE`：强制 PWM 关闭，清检查和角度状态。
- `SENSOR_CHECK`：确认 MA600、当前电流和 PWM off safe。
- `ALIGN`：运行 `Motor_LoopRunAlign()`，完成 PWM 环境零漂校准和闭环前转子对齐。
- `CLOSED_LOOP`：根据控制模式允许闭环或开环输出。
- `FAULT`：强制关闭 PWM。

`Motor_SetEnable(0)` 会：

```text
s_enable = 0
s_ctrl_mode = MOTOR_CTRL_OFF
ClearFault()
s_state = MOTOR_STATE_IDLE
Board_ForcePwmOff()
ClearCheck()
ClearAngle()
Motor_LoopInit()
```

## 6. 闭环控制

`Motor_Loop.c` 当前只保留正式闭环主线：

```text
ReadCurrent()
SnapshotAngleToLoop()
MA600/角度/电流安全检查
Clarke
Park
电流参考选择
PI(id), PI(iq)
Foc_LimitDq()
InvPark
SVPWM
Board_SetPwmDuty()
Board_EnablePwmOutput()
```

控制模式：

- `MOTOR_CTRL_CURRENT`：直接使用 `g_cmd.current.id_ref/iq_ref`。
- `MOTOR_CTRL_SPEED`：速度环输出 `iq_ref`，再进入电流环。
- `MOTOR_CTRL_OFF`：不输出。
- `MOTOR_CTRL_VF / MOTOR_CTRL_IF`：转交 `Motor_OpenLoop`。

速度估算当前来自 MA600 累计位置差分：

```text
pos_delta = Motor_GetAnglePos() - prev_pos
fb_raw = pos_delta * MOTOR_SENSOR_DIR * MOTOR_SPEED_EST_HZ
fb_filt += (fb_raw - fb_filt) >> MOTOR_SPEED_FILTER_SHIFT
```

## 7. 开环检测

`Motor_OpenLoop.c/h` 是保留的正式检测辅助，不属于临时测试钉子。

用途：

- `MOTOR_CTRL_VF`：开环电角度 + 固定 d 轴电压，用于看 PWM 三相变化、电流采样周期性和相序趋势。
- `MOTOR_CTRL_IF`：开环电角度 + 电流 PI，用于检查电流方向和采样方向。

命令入口：

```text
g_cmd.run.ctrl_mode = MOTOR_CTRL_VF 或 MOTOR_CTRL_IF
g_cmd.open_loop.ol_speed_ref
g_cmd.open_loop.vf_voltage
g_cmd.open_loop.if_id_ref
g_cmd.open_loop.if_iq_ref
g_cmd.open_loop.ol_timeout_ms
```

观察入口：

```text
g_elementary.open_loop.theta
g_elementary.open_loop.vf_voltage
g_elementary.open_loop.if_id_ref
g_elementary.open_loop.if_iq_ref
g_elementary.open_loop.current_over_count
```

## 8. FOC 数学

`Motor_Foc.c/h` 提供：

- `Foc_Sin/Foc_Cos`
- `Foc_ClarkeTransform`
- `Foc_ParkTransform`
- `Foc_InvParkTransform`
- `Foc_PiUpdate`
- `Foc_LimitDq`
- `Foc_Svpwm`

当前仍是定点整数实现，适合 CMS32M6513AGE40NB 这种无 FPU 的 Cortex-M0+。后续优化方向：

- 减少快环除法。
- 明确电流 count 到实际 A 的换算。
- 根据实测响应重新整定 PI。
- 需要时把 FOC 中常用短函数改为 `static inline`。

可写命令：

```text

g_cmd.run.ctrl_mode

g_cmd.speed.speed_ref
g_cmd.speed.iq_limit
g_cmd.angle.elec_zero_trim

```

只读观察：

```text
g_elementary.motor

g_elementary.foc
g_elementary.pwm
g_elementary.speed
g_elementary.safety

```

原则：

- Watch/JScope 只写 `g_cmd`。
- Watch/JScope 默认只读 `g_elementary`。
- 不再新增散乱全局变量。

## 10. 当前风险和后续改进

当前仍需注意：

1. 电流环 PI 仍是 bring-up 参数，不能当成最终参数。
2. 当前 MA600 在 ADC 中断内 SPI 读角，快环提高前必须评估耗时。
3. `App_Debug` 快照不是严格原子快照，JScope 调试可以接受，严肃数据记录后续要做一次性复制接口。
4. `Motor` 仍直接依赖 `Board_Analog.h` 和 `Board_PWM.h`，短期可接受，后续可改为 Board 统一快照。
5. 位置环尚未实现，接回丝杠前必须加软限位、回零、最大速度/加速度和跟随误差保护。

当前不建议再恢复：

- `App_Test`
- `Motor_Diag`
- `debug_*`
- `g_run/g_debug/g_test`
- MA600 4 字节诊断常驻入口
- ADC 采样窗口统计测试程序
